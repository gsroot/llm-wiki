# 2026 육아휴직 1년 운영 계획

> [!important] 실행형 hub — 시간 예산이 박힌 1년 운영 계획
> 신설 (Codex 권고 P1). [[seokgeun-operating-profile-2026|운영 프로필]]이 정의한 우선순위(가족·MateChat 사업화·번아웃 방지)를 시간 단위 실행 계획으로 변환. 월간 회고로 갱신 (`verification_required: true`).
> 
> 한국어 표기: **육아휴직 1년 운영 계획**.

## 언제 읽어야 하는가

- "육아휴직 중 어떻게 시간을 배분하지?" — 시간 예산표 직행.
- "MateChat 출시·검증 일정과 가족 시간이 충돌하면?" — 충돌 해소 원칙 섹션.
- "번아웃 신호가 보이면 무엇을 멈춰야 하나?" — 비상 정지 트리거 섹션.
- "월말 회고 시 무엇을 회수하는가?" — 회수 체크리스트.

## 1. 1년 시간 예산 (주 단위)

| 영역 | 주당 시간 | 비고 |
|---|---|---|
| 가족 (육아·식사·동행) | 40~50h | 최우선. 다른 모든 항목보다 위. |
| MateChat 운영 (출시·CS·이터레이션) | 15~20h | closed alpha 기간엔 +5h 가능 |
| 위키·portfolio 운영 | 5~8h | 일 1h 평균, 주기 단위 |
| AI/Agent 학습 (5축 LLM 인프라) | 5~10h | 사이드 학습. 회사 BI 차용 검토 포함 |
| 신체·정신 건강 (운동·수면·취미) | 10~15h | **번아웃 방지의 일차 방어선** |
| 사회적 관계 (친구·동료) | 3~5h | 분기별 회복 |
| **합계** | 78~108h | 주 168h 중 실제 가용 ~110h |

핵심: **가족이 절대 최상위**. 나머지 항목은 가족 시간을 침범하지 않는 선에서 우선순위 조정.

## 2. 월간 마일스톤 (분기별)

### Q1 (2026-01~03): MateChat 출시 + 운영 안정화
- v1.0.0 Google Play 정식 출시 (QA 완료 후)
- closed alpha 10명 모집 + 첫 30일 검증 (→ [[matechat-30day-validation-loop]])
- 가족: 신생아 적응 (만약 해당), 매주 가족 동행 1회 보장

### Q2 (2026-04~06): MateChat KPI 검증 + 회사 차용 후보 탐색
- D30 retention 측정값 회수
- MateChat 자작 SKILL 9개 중 c2spf 차용 후보 1건 PoC (→ [[c2spf-ai-agent-adoption-candidates]])
- 가족: 첫 가족 여행 (국내)

### Q3 (2026-07~09): 사업화 의사결정 분기점
- D90 retention + paying user 수 검증
- "MateChat을 사업으로 키울 것인가" 의사결정 (Go/No-Go)
- Go → v1.1 로드맵 + 자금 계획 / No-Go → 출시 후 minimal 운영 + 다음 사이드 후보 탐색
- 가족: 분기별 휴가 1주

### Q4 (2026-10~12): 복직 준비 + 회수
- 회사 복귀 계획 정립
- 1년 회고 작성 (회수 체크리스트 기반)
- MateChat·portfolio·위키 자료 정리
- 가족: 연말 휴식

## 3. 주간 리듬

| 요일 | 오전 (3~4h) | 오후 (3~4h) | 저녁 |
|---|---|---|---|
| 월 | 가족 / 신체 | MateChat 운영 | 가족 |
| 화 | MateChat 이터레이션 | 위키 갱신 | 가족 |
| 수 | 가족 / 외출 | AI 학습 | 가족 |

| 일 | 신체·취미 | 가족 | 다음 주 계획 |

## 4. 충돌 해소 원칙

상위 항목이 하위를 항상 이긴다:

1. **가족 응급** (질병·요청·동행) → 모든 일정 중단
2. **신체·정신 건강 위험 신호** → MateChat·위키·학습 모두 중단, 1주 회복
3. **MateChat 출시 차단 이슈** → 위키·학습 우선순위 낮춤
4. **MateChat 운영 (CS·버그 수정)** → 시점 순연 가능
5. **위키·portfolio 운영** → 학습 시간을 빌려 처리
6. **AI/Agent 학습** → 가용 시간 내에서만

## 5. 비상 정지 트리거 (번아웃 방지)

다음 신호 1개 이상 감지 시 모든 사이드 작업 24시간 정지 + 가족·휴식만:

- 수면 5시간 미만이 3일 연속
- 가족과의 짧은 대화에서 짜증·방어적 반응이 3회 이상
- "이 작업을 왜 하지?"라는 회의가 2일 연속
- 신체 증상(두통·소화불량·어깨 통증)이 1주 지속

회복 후에도 동일 작업으로 즉시 복귀 금지 — 1주는 **다른 영역**(예: 위키 → 신체 운동)으로 전환.

## 6. 월간 회수 체크리스트

월말마다 다음 항목을 [[seokgeun-operating-profile-2026]] 또는 본 페이지에 갱신:

- [ ] MateChat KPI 측정값 (paying users / D30 / 인간-인간 메시지 5회 비율)
- [ ] c2spf 차용 PoC 진행 상태 (검토/진행/완료/포기)
- [ ] 가족 시간 실측 vs 계획 (괴리 시 원인)
- [ ] 번아웃 신호 발생 횟수 + 정지 트리거 발동 여부
- [ ] 시점 진행 (월간 진행)
- [ ] 다음 달 1개 핵심 의사결정 (예: "v1.1 시작 vs 검증 추가 1개월")
