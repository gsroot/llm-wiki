## 사용 가이드

- **특정 회차 빠르게 찾기**: 본 색인에서 회차 행을 찾아 "점프 →" 클릭 → [[log]] 해당 섹션으로 이동.
- **시간순 연속 회독**: [[log]]를 직접 읽음 (전체 2,088줄, 회차 간 맥락 보존됨).
- **최근 5회차 회고**: [[log]] 상단 5개 ## 섹션이 항상 최신 회차.
- **회차별 작업 종류**: 제목 컬럼의 prefix(`docs`/`infra`/`ingest`/`cleanup`/`fix`/`synthesis`/`lint` 등)가 작업 카테고리를 표시.

## 메타

본 색인은 45회차에 신설됐다. 이전까지는 [[log]] 단일 파일을 Ctrl+F로 검색해야 했고, 길이가 2,000줄을 넘어 O(n) 탐색이 불편하다는 평가가 43회차에 제기됐다. log.md 자체를 분할하지 않고 색인만 신설한 이유:

1. **위키링크 호환성**: `[[log]]` 인용을 변경 없이 보존 (3건 인바운드)
2. **시간순 맥락 보존**: 회차 간 흐름은 단일 파일에서만 자연스러움
3. **자동 생성 가능**: log.md의 ## 헤더만 정규식으로 추출하면 색인 자동 갱신 (스크립트 1회 실행)

향후 log.md가 5,000줄을 넘으면 archive 분할(`log-archive-202604.md` 등)을 검토한다. 현재는 색인 추가로 충분.
