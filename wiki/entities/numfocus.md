# NumFOCUS

## 개요

**NumFOCUS**는 과학 컴퓨팅·데이터 분석 오픈소스 프로젝트를 후원하는 501(c)(3) 비영리 조직이다. [[pandas]]와 [[scikit-learn]] 같은 PyData 생태계 핵심 프로젝트가 NumFOCUS sponsored project로 운영된다.

위키에서 중요한 점은 NumFOCUS가 단순 스폰서가 아니라 **자금·법무·상표·후원 수금 같은 비기술 운영을 맡는 거버넌스 층**이라는 점이다. [[pandas-dev]] 사례에서는 기술 의사결정은 Core Team/BDFL/PDEP가 맡고, NumFOCUS Subcommittee는 자금 관리에 한정된다.

## 주요 특징

- **비영리 후원 구조**: 기업 후원을 프로젝트에 전달하고 회계·법무 부담을 줄임.
- **기술-자금 분리**: pandas는 NumFOCUS가 자금을 관리하지만 기술 방향에 직접 관여하지 않도록 명시한다.
- **PyData 허브**: pandas, scikit-learn, NumPy/Jupyter 계열처럼 Python 데이터 생태계의 장기 안정성을 떠받치는 운영 기반.

## 왜 중요한가

LLM 위키 관점에서는 "좋은 오픈소스가 왜 오래 가는가"를 설명하는 조직적 답이다. [[scikit-learn]]의 19년 API 안정성이나 [[pandas]]의 PDEP 기반 거버넌스는 코드 품질만으로 유지된 것이 아니라, 비영리 후원·풀타임 개발자·기업 후원·커뮤니티 결정 구조가 결합된 결과다.

## 관련 개념

- [[pandas-dev]]: pandas 조직의 후원·거버넌스 맥락.
- [[pandas]]: 2015년 NumFOCUS sponsored project로 공식화.
- [[scikit-learn]]: INRIA·probabl.ai·NumFOCUS·기업 후원이 겹친 안정적 운영 사례.
- [[bdfl]] / [[pdep]]: pandas의 기술 의사결정 층. NumFOCUS와 역할이 분리된다.

## 논쟁/모순

> [!warning] 논쟁/모순
> 현재 위키 안에서 NumFOCUS 관련 모순은 없음. 다만 프로젝트별로 NumFOCUS가 맡는 범위는 다를 수 있으므로, 다른 PyData 프로젝트를 수집할 때 기술 의사결정권과 자금 관리권을 분리해서 기록해야 한다.
