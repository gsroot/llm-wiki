# 레슨 제목

> **시간**: 약 30분 · **난이도**: beginner
> **사전 지식**: [[관련-개념]]을 먼저 읽으면 도움이 됩니다.
> **이 레슨을 끝내면**: ① ... ② ... ③ ... 할 수 있습니다.

## 1. Warmup — 학습 전 자가 점검

> 본문 읽기 전, 다음 질문에 답해보세요. 이미 아는 내용을 확인하고, 모르는 부분에 호기심을 유발합니다.

1. **질문**: ...
   - 후보: A. ... / B. ... / C. ... (정답은 본문 마무리에서 밝힘)
2. **질문**: ...
3. **질문**: ...

## 2. 본문

### 핵심 개념

이 레슨이 다루는 개념의 정의와 작동 원리를 설명합니다. 본문은 짧고 강하게 — 한 화면에 들어오도록.

### 단계별 빌드 / 시연

프로젝트 기반 lesson이라면 단계별 가이드:

1. **단계 1**: ...
   ```language
   # 코드 예제
   ```
2. **단계 2**: ...
3. **단계 3**: ...

### 시각 자료 (선택)

다이어그램·sketchnote·그래프가 있으면 여기에. 이미지는 `raw/assets/`에 저장 후 상대 경로로 참조.

## 3. Knowledge Check — 중간 점검

> 본문에서 가장 중요한 3가지를 확인합니다. 즉답하지 못하면 본문으로 돌아가세요.

- [ ] **질문 1**: ... ?
- [ ] **질문 2**: ... ?
- [ ] **질문 3**: ... ?

## 4. Challenge — 도전 과제

> 본문 범위를 살짝 넘는 응용 과제. 정답이 한 가지가 아닌 **개방형**으로 둡니다.

> 예: "본문에서 다룬 패턴을 ___ 상황에 적용해본다면 어떻게 변형해야 할까?"

## 5. 추가 자료 — Supplemental Reading

레슨 범위 밖이지만 흥미로운 후속 자료:

- [[관련-소스]] — 한 줄 설명
- 외부 링크: [공식 문서](https://...) — 한 줄 설명
- 영상: ... (시간 N분)

## 6. Assignment — 실습 과제

> 학습 내용을 자기 문제에 적용. 결과물을 어디에 남길지(`projects/`, github repo, 위키 [[종합 분석]])도 함께 정합니다.

**과제**:
- 입력: ...
- 작업: ...
- 결과물: ...
- 마감: 자율 (예: 다음 주말)

**자기 채점 루브릭**:
- [ ] 핵심 개념을 코드/문서로 표현했는가?
- [ ] 실패 케이스 1가지 이상 다뤘는가?
- [ ] 기존 위키 페이지([[...]])와 연결했는가?

## 7. Wrap-up — 학습 후 자가 점검

> Warmup의 답을 다시 확인 + 본문 흡수 점검.

1. **Warmup 1번 정답**: ... (왜 그런지 본문 ___ 섹션 참조)
2. **새로 알게 된 것**: ...
3. **여전히 모호한 것**: ... (→ [[열린 질문]] 섹션에 추가)
