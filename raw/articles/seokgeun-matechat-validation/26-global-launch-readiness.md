# 26. 글로벌 출시 준비 상태 진단

> **작성일:** 2026-03-19
> **목적:** 전 세계 사용자가 Mate Chat을 통해 채팅할 수 있는지 점검하고, 글로벌 출시 전 해결해야 할 장애물을 정리한다.
> **상태:** 진단 완료, 작업 미착수

---

## 1. 진단 요약

| 등급 | 항목 수 | 설명 |
|------|---------|------|
| CRITICAL | 3 | 출시 불가 — 반드시 수정 필요 |
| HIGH | 2 | 출시 품질에 심각한 영향 |
| MEDIUM | 1 | 사소한 UX 문제 |

**결론:** 기술 아키텍처(DB, WebSocket, 인코딩)는 글로벌 채팅에 적합하지만, 결제 보안 구멍·한국어 하드코딩·콘텐츠 모더레이션 부재·법적 문서 미비로 실질적 글로벌 출시는 불가.

## 2. CRITICAL — 출시 차단 이슈

### 2.1 인앱 결제(IAP) 검증이 mock 상태

**파일:** `mate_chat_backend/app/services/clover_service.py`

**현황:**
- `verify_google_purchase()`와 `verify_apple_purchase()` 모두 `mock_verification_success = True`를 반환
- 실제 스토어 서버와의 영수증 검증이 구현되지 않음

**영향:**
- 사용자가 결제 없이 무한 클로버 획득 가능 (매출 손실 + 사기)
- Apple App Store 심사에서 리젝 사유
- Google Play 정책 위반

**수정 방안:**
1. Google Play Developer API 연동 (`google-api-python-client`)
   - `purchases.products.get()` 또는 `purchases.subscriptions.get()` 호출
   - 영수증의 `purchaseToken`을 서버 측에서 검증
2. Apple StoreKit 2 Server API 연동
   - App Store Server API v2 (`/inApps/v2/transactions/{transactionId}`)
   - JWS(JSON Web Signature) 기반 서버 알림 수신
3. 이중 지출 방지: 검증된 영수증의 `orderId`/`transactionId`를 DB에 저장하여 중복 사용 차단

**예상 소요:** 3~5일

---

### 2.2 푸시 알림 한국어 하드코딩

**파일:**
- 백엔드: `mate_chat_backend/app/services/notification_service.py`
- Flutter: `mate_chat_flutter/lib/core/push/push_notification_service.dart`

**현황:**
```python
# 백엔드 예시 (notification_service.py)
"title": f"{sender.nickname}님의 새 메시지"  # 한국어 고정
```
```dart
// Flutter 예시 (push_notification_service.dart)
case 'chat':
  return sender != null ? '$sender님의 새 메시지' : '새 메시지';  // 한국어 고정
case 'invite':
  return '채팅방 초대';  // 한국어 고정
```
- Android 알림 채널명도 `'Mate Chat 알림'`으로 한국어 고정

**영향:**
- 비한국어 사용자(전체 잠재 사용자의 ~88%)가 알림 내용을 읽을 수 없음
- 앱 내 UI는 영어(en) fallback 정책으로 비 한국어 locale에 영어가 노출되지만, 푸시 알림은 서버 발송 시점에 사용자 locale 정보가 없어 한국어로 발송될 수 있음 → 동일한 en fallback 정책을 서버 알림에도 적용해야 함
- 앱의 핵심 기능인 채팅 알림이 무의미해짐 → 사용자 이탈

**수정 방안:**
1. `device_tokens` 테이블에 `locale` 컬럼 추가 (사용자 기기 언어)
2. 백엔드: 알림 생성 시 수신자의 locale에 따라 번역된 템플릿 사용
3. Flutter: 하드코딩된 한국어 문자열을 `context.l10n` 기반으로 교체 (사용자 노출 fallback 언어는 영어(en)이며, 한국어 문자열은 `ko` locale에서만 노출)
4. 알림 템플릿 파일을 별도 관리 (JSON 또는 ARB 형식)

**예상 소요:** 2~3일

### 2.3 콘텐츠 모더레이션 부재

**현황:**
- 채팅 메시지, 사용자 프로필, AI 대화에 대한 필터링/검수 로직이 전혀 없음
- `user_reports` 테이블은 존재하지만, 신고 처리 워크플로우 미구현
- 스팸 감지, 욕설 필터, 부적절 콘텐츠 차단 기능 없음

**영향:**
- 법적 책임: UGC(사용자 생성 콘텐츠) 미관리 시 각국 법률 위반 가능
  - EU: Digital Services Act (DSA) — 불법 콘텐츠 신속 제거 의무
  - 한국: 정보통신망법 — 음란·폭력 콘텐츠 차단 의무
  - 미국: COPPA — 13세 미만 보호
- 플랫폼 악용: 스팸, 피싱, 혐오 발언 등에 무방비
- 앱스토어 정책 위반으로 삭제 가능

**수정 방안 (단계적):**

**Phase 1 — 최소 보호 (출시 전 필수, 5~7일)**
1. 키워드 기반 욕설/혐오 필터
   - 다국어 금칙어 사전 (한국어, 영어, 일본어, 중국어 등)
   - 메시지 전송 시 필터 적용 → 경고 또는 차단
2. 스팸 감지
   - 동일 메시지 반복 감지 (N초 내 동일 내용 M회)
   - 과도한 메시지 속도 제한
3. 신고 처리 기본 워크플로우
   - 신고 N건 누적 시 자동 임시 차단
   - 관리자 대시보드 (기본)

**Phase 2 — 고급 모더레이션 (출시 후 개선)**
1. AI 기반 콘텐츠 분류 (OpenAI Moderation API 또는 Perspective API)
2. 이미지/미디어 검사 (향후 미디어 공유 기능 추가 시)
3. 사용자 신뢰도 점수 시스템

## 3. HIGH — 출시 품질에 심각한 영향

### 3.1 UI 번역 — 네이티브 커버리지 부족 (영어 fallback으로 방어)

**현황:**
- i18n 인프라 구축 완료: 9개 로케일 (ko, en, ja, zh-Hans, zh-Hant, es, id, vi, pt-BR)
- `app_ko.arb` 952개 키 추출 완료, `app_en.arb`도 952개 키 전량 영어로 커버
- 나머지 7개 locale은 locale당 평균 `376`개 키만 네이티브 번역 → 576개 키는 빌드 파이프라인에서 영어(en)로 자동 주입
- **결과적으로 사용자에게 한국어가 노출되는 일은 없으나, 네이티브 번역 커버리지가 낮아 UX 품질 리스크가 있음**

**하드코딩 문자열 분포 (과거 스냅샷, 대부분 ARB로 이관 완료):**

| 영역 | 과거 미번역 문자열 수 | 주요 파일 |
|------|---------------------|-----------|
| 채팅 기능 | ~244 | `chat_page.dart`, `chat_room_*.dart` |
| 설정 | ~197 | `settings_page.dart`, `blocked_users_page.dart` |
| 챗봇 | ~166 | `chatbot_*.dart` |
| 인증 | ~145 | `login_page.dart`, `register_page.dart` |
| 소셜 | ~123 | `mate_requests_page.dart`, `followers_page.dart` |
| 프로필 | ~110 | `profile_*.dart`, `edit_profile_page.dart` |
| 결제 | ~46 | `purchase_page.dart` |

**영향:**
- 비한국어 사용자에게 한국어는 노출되지 않음(영어 fallback으로 방어)
- 다만 네이티브 번역 커버리지가 낮아 일본어/중국어/스페인어/인니어/베트남어/브라질 포르투갈어 사용자는 일부 화면에서 영어가 섞여 보이는 혼합 UX 발생
- 앱스토어 리뷰에서 “부분 번역” 관련 부정 평가 가능

**수정 방안:**
1. 각 locale ARB의 네이티브 번역 커버리지를 952개 키까지 끌어올린다(우선순위: `ja`, `zh_Hans`, `en` 검수 → 기타).
2. 잔존 하드코딩 문자열이 있다면 `context.l10n.키` 형태로 이관한다.
3. 전문 번역 서비스 또는 AI 번역 활용 (초벌 → 네이티브 검수).
4. RTL 언어(아랍어 등)는 현재 제외 — 레이아웃 대응 비용이 높음.

**참고 문서:** `docs/24-flutter-i18n-preparation.md`, `docs/25-localization-glossary.md`

**예상 소요:** 2~3주

### 3.2 법적 문서 및 규정 준수 미비

**현황:**
- 이용약관(Terms of Service) 없음
- 개인정보처리방침(Privacy Policy) 없음
- 쿠키 동의 배너 없음
- GDPR 데이터 내보내기(Data Portability) 미구현
- COPPA 연령 확인 미구현
- `DELETE /v1/users/me`는 존재하지만 소프트 삭제만 수행 (GDPR "잊힐 권리" 미완전 대응)

**영향:**
- EU 출시 불가 (GDPR 필수 요건 미충족)
- 미국 13세 미만 사용자 보호 미비 (COPPA 위반)
- Apple/Google 앱스토어 심사 시 개인정보처리방침 필수

**수정 방안:**
1. 법적 문서 작성
   - 이용약관: 사용자 행동 규칙, 콘텐츠 정책, 면책 조항
   - 개인정보처리방침: 수집 데이터, 사용 목적, 보관 기간, 권리 행사 방법
2. GDPR 대응
   - 데이터 내보내기 API (`GET /v1/users/me/data-export`)
   - 삭제 요청 시 관련 데이터 완전 삭제 또는 익명화
   - 처리 동의 기록 관리
3. COPPA 대응
   - 회원가입 시 생년월일 수집 → 13세 미만 가입 차단
4. 앱 내 법적 문서 표시
   - 회원가입 시 동의 체크박스
   - 설정 > 법적 정보 페이지

**예상 소요:** 1~2주

## 4. MEDIUM — 사소한 UX 문제

### 4.1 일부 페이지의 시간대 표시 오류

**파일:**
- `mate_chat_flutter/lib/features/*/presentation/` 내 일부 페이지

**현황:**
- 백엔드는 모든 타임스탬프를 UTC로 정확히 저장
- Flutter에서 `.toLocal()` 사용 시 일부 페이지에서 올바른 시간대 변환이 누락
- 상대 시간("2시간 전")은 정상, 절대 시간(날짜/시간 표시)에서 ±12시간 오차 가능

**영향:** 경미 — 메시지 시간 표시가 일부 상황에서 부정확할 수 있음

**수정 방안:** `date_formatter.dart`의 유틸리티 함수를 모든 페이지에서 일관되게 사용

**예상 소요:** 1~2일

## 5. OK — 글로벌 대응 완료 영역

### 5.1 유니코드/이모지 지원 ✅

- PostgreSQL: UTF-8 인코딩
- 모든 텍스트 필드: SQLAlchemy `Text` 타입 (길이 제한 없음)
- Dart 3.10.1+: 네이티브 유니코드 지원
- CJK 문자, 아랍어, 이모지, 특수문자 모두 정상 처리

### 5.2 데이터베이스 스키마 국제화 ✅

- `users.languages: List[str]` — 다중 언어 설정
- `users.country_code: String(3)` — ISO 3166-1 국가 코드
- 닉네임, 소개, 채팅방 이름 등: 무제한 UTF-8 텍스트
- 모든 타임스탬프: timezone-aware UTC

### 5.3 WebSocket 아키텍처 ✅

- 지수 백오프 재연결 (1s → 2s → 4s → 8s → max 30s)
- 오프라인 메시지 큐잉
- 30초 하트비트 (ping/pong)
- 토큰 만료 시 자동 갱신 후 재연결
- Redis Pub/Sub 구조로 다중 서버 확장 가능 (구현 완료, 배포 대기)

### 5.4 네트워크 복원력 ✅

- WebSocket 자동 재연결으로 불안정한 네트워크 대응
- HTTP 요청 실패 시 UI에 에러 표시
- 개발도상국 저속 네트워크에서도 기본 동작 가능

### 5.5 사용자 탐색/매칭 ✅

- 국가, 언어, 성별, 나이 필터로 사용자 검색 가능
- 다국어 태그 시스템
- 글로벌 사용자 간 1:1 및 그룹 채팅 지원

## 7. 참고 사항

### 현재 제외된 범위

- **RTL 언어 지원** (아랍어, 히브리어): Flutter 레이아웃 전면 수정 필요, 1차 출시에서 제외 권장
- **지역별 서버 배포**: 현재 단일 리전 구조로도 기능 동작. 사용자 증가 시 CDN/멀티 리전 검토
- **현지 결제 수단**: Google Play/App Store IAP로 대부분의 국가 커버 가능

### 관련 문서
