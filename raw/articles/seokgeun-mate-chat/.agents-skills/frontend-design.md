# UI/UX 디자인

일반적인 "AI스러운" 미학을 피하고 독특하고 프로덕션급의 UI/UX 인터페이스를 생성하기 위한 가이드입니다.
웹(HTML/CSS/JS, React, Vue)과 모바일(Flutter, React Native) 모두에 적용됩니다.
미학적 디테일과 창의적 선택에 대한 탁월한 주의를 기울여 실제 작동하는 코드를 구현합니다.

사용자는 UI/UX 요구사항을 제공합니다: 구축할 컴포넌트, 페이지, 화면, 애플리케이션 또는 인터페이스.
목적, 대상 또는 기술적 제약사항에 대한 컨텍스트를 포함할 수 있습니다.

---

## 관련 시스템

**Slash Command:** `/ui` (사용자 명시적 호출)
- 사용자가 `/ui 프로필 화면 디자인 개선` 형식으로 UI 작업 시작
- Flutter UI/UX 개선 및 디자인 적용 작업을 명시적으로 시작

**Sub-Agent:** `flutter-ui-ux-expert` (자동 호출)
- UI 디자인, 애니메이션, 스타일링 전문가 에이전트
- Material Design 3, 커스텀 애니메이션, 반응형 레이아웃 전문가
- 트리거 조건: UI 미화, 애니메이션 추가, 테마 커스터마이징, 디자인 개선

**Agent Skill:** `frontend-design` (이 스킬)
- 웹과 모바일 UI 디자인 가이드라인 제공
- 독특한 미학, 창의적 선택, 프로덕션급 인터페이스 설계 원칙

**관련 스킬:**
- `theme-factory`: 색상 테마 및 폰트 적용
- `flutter-patterns`: Flutter UI 구조 및 패턴 준수
- `web-artifacts-builder`, `flutter-artifacts-builder`: 프로젝트 초기화 시 적용

## 디자인 사고

코딩 전에 컨텍스트를 이해하고 대담한 미학적 방향을 설정합니다:

- **목적**: 이 인터페이스가 해결하는 문제는 무엇인가? 누가 사용하는가?
- **톤**: 극단을 선택: 극도로 미니멀, 최대주의적 혼돈, 레트로 퓨처리스틱, 유기적/자연적, 럭셔리/세련된, 장난스러운/장난감 같은, 에디토리얼/잡지, 브루탈리스트/원시적, 아르데코/기하학적, 부드러운/파스텔, 산업적/실용적 등. 선택할 수 있는 다양한 풍미가 있습니다. 이것들을 영감으로 사용하되 미학적 방향에 충실한 디자인을 만듭니다.
- **제약사항**: 기술적 요구사항 (프레임워크, 성능, 접근성).
- **차별화**: 무엇이 이것을 잊을 수 없게 만드는가? 사람들이 기억할 한 가지는 무엇인가?

**중요**: 명확한 개념적 방향을 선택하고 정밀하게 실행합니다. 대담한 최대주의와 세련된 미니멀리즘 모두 효과적입니다 - 핵심은 강도가 아닌 의도성입니다.

그런 다음 다음과 같은 작동하는 코드를 구현합니다:
- **웹**: HTML/CSS/JS, React, Vue, Svelte 등
- **모바일**: Flutter, React Native 등
- 프로덕션급이고 기능적
- 시각적으로 인상적이고 기억에 남음
- 명확한 미학적 관점을 가진 응집력
- 모든 디테일이 세심하게 다듬어짐

## UI/UX 미학 가이드라인

### 공통 원칙 (웹 & 모바일)

다음에 집중:

- **타이포그래피**: 아름답고 독특하며 흥미로운 폰트를 선택합니다. Arial이나 Inter 같은 일반적인 폰트를 피하고, 프론트엔드의 미학을 높이는 독특한 선택을 선호합니다. 예상치 못한 개성 있는 폰트 선택. 독특한 디스플레이 폰트와 세련된 본문 폰트를 페어링합니다.

- **색상 및 테마**: 응집력 있는 미학에 전념합니다. 일관성을 위해 CSS 변수를 사용합니다. 날카로운 악센트가 있는 지배적인 색상이 소심하고 균등하게 분산된 팔레트보다 우수합니다.

- **모션**: 효과와 마이크로 인터랙션을 위해 애니메이션을 사용합니다. HTML의 경우 CSS 전용 솔루션을 우선시합니다. React의 경우 가능하면 Motion 라이브러리를 사용합니다. 영향력이 큰 순간에 집중: 시차를 둔 reveal(animation-delay)을 가진 잘 조율된 페이지 로드가 흩어진 마이크로 인터랙션보다 더 큰 기쁨을 만듭니다. 스크롤 트리거와 놀라운 호버 상태를 사용합니다.

- **공간 구성**: 예상치 못한 레이아웃. 비대칭. 겹침. 대각선 흐름. 그리드를 깨는 요소. 여유로운 네거티브 스페이스 또는 제어된 밀도.

- **배경 및 시각적 디테일**: 단색 배경 대신 분위기와 깊이를 만듭니다. 전체 미학과 일치하는 맥락적 효과와 텍스처를 추가합니다. 그래디언트 메시, 노이즈 텍스처, 기하학적 패턴, 레이어드 투명도, 극적인 그림자, 장식적 테두리, 커스텀 커서, 그레인 오버레이와 같은 창의적인 형태를 적용합니다.

절대 사용하지 말아야 할 일반적인 AI 생성 미학:
- 과도하게 사용된 폰트 패밀리 (Inter, Roboto, Arial, 시스템 폰트)
- 진부한 색 구성 (특히 흰색 배경에 보라색 그래디언트)
- 예측 가능한 레이아웃과 컴포넌트 패턴
- 컨텍스트별 개성이 부족한 쿠키커터 디자인

창의적으로 해석하고 컨텍스트에 진정으로 디자인된 느낌을 주는 예상치 못한 선택을 합니다. 어떤 디자인도 동일해서는 안 됩니다. 라이트와 다크 테마, 다른 폰트, 다른 미학 사이를 다양하게 조합합니다. 세대 간 일반적인 선택 (예: Space Grotesk)으로 수렴하지 마십시오.

**중요**: 구현 복잡도를 미학적 비전에 맞춥니다. 최대주의 디자인은 광범위한 애니메이션과 효과를 가진 정교한 코드가 필요합니다. 미니멀리스트 또는 세련된 디자인은 절제, 정밀성, 간격, 타이포그래피 및 미묘한 디테일에 대한 신중한 주의가 필요합니다. 우아함은 비전을 잘 실행하는 데서 나옵니다.

### Flutter/모바일 특화 가이드라인

모바일 앱 UI를 디자인할 때 추가로 고려:

- **타이포그래피**:
  - Google Fonts 또는 커스텀 폰트 사용
  - Pretendard, Noto Sans 같은 한국어 최적화 폰트 고려
  - 플랫폼 가이드라인 존중하되 독창성 유지

- **색상 및 테마**:
  - Material Design 3 또는 Cupertino 스타일 활용
  - 다크/라이트 모드 지원
  - 플랫폼별 색상 시스템 (Material: ColorScheme, iOS: CupertinoColors)

- **모션 및 애니메이션**:
  - Flutter의 Hero 애니메이션, AnimatedContainer, Tween 활용
  - 플랫폼 네이티브 느낌의 트랜지션
  - 60fps 유지를 위한 성능 최적화

- **공간 구성**:
  - SafeArea 고려 (노치, 하단 바)
  - 터치 타겟 크기: 최소 48x48
  - 반응형 레이아웃 (MediaQuery, LayoutBuilder)
  - 스크롤 동작과 오버스크롤 효과

- **플랫폼 적응**:
  - iOS/Android 플랫폼별 디자인 언어 존중
  - Material 위젯 vs Cupertino 위젯
  - 플랫폼별 네비게이션 패턴
  - 접근성: Semantics 위젯 사용

절대 사용하지 말아야 할 모바일 AI 미학:
- 지나치게 둥근 모서리와 그림자 남용
- 기본 Material/Cupertino 테마를 그대로 사용
- 일반적인 그래디언트 (보라색-파란색)
- 과도한 애니메이션으로 성능 저하

### 웹 특화 가이드라인

웹 인터페이스를 디자인할 때:

- **모션**: HTML의 경우 CSS 전용 솔루션 우선. React의 경우 Motion 라이브러리 사용.
- **반응형**: 브레이크포인트, 유동적 그리드, 컨테이너 쿼리
- **접근성**: ARIA 레이블, 키보드 네비게이션, 스크린 리더 지원
- **성능**: 레이지 로딩, 코드 스플리팅, 최적화된 이미지
