# Chapter 14. Codex로 코드 작성과 리팩토링

## 페이지 구조 (헤딩 outline)
- Codex에 태스크 할당하는 방법
- 코드 작성 태스크 유형
  - 새로운 기능 구현
  - 기존 코드 리팩토링
  - 코드 변환 (마이그레이션)
- 효과적인 태스크 작성 팁
  - 팁 1: 코딩 컨벤션을 AGENTS.md에 위임하라
  - 팁 2: 완료 기준을 구체적으로 명시하라
  - 팁 3: 단계적으로 태스크를 분할하라
- Codex 작업 결과 리뷰 방법
- 실습: Flask REST API에 CRUD 엔드포인트 추가하기

## 핵심 요약 (자가 작문)

14장은 Codex에 코딩 작업을 효과적으로 위임하기 위한 프롬프트 작성 방법론에 집중한다. 좋은 태스크는 목표(무엇을), 기술 스택(어떻게), 완료 기준(검증 방법) 세 요소를 모두 갖춰야 Codex가 일관된 결과를 낸다. 코딩 컨벤션처럼 반복되는 규칙은 매번 프롬프트에 붙이지 말고 AGENTS.md에 한 번 정의해 위임하는 편이 안정적이며, 큰 기능은 작은 단위로 쪼개 순차 실행해야 실패 시 디버깅이 쉽다. 작업 결과는 PR diff·테스트 통과 여부·로그를 사람이 직접 검토해야 하며, 실습은 Flask REST API에 CRUD 엔드포인트를 추가하는 흐름으로 위 원칙을 적용한다.
