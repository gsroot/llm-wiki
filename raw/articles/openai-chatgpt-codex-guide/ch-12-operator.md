## 핵심 요약 (자가 작문)

Operator는 OpenAI가 선보인 Computer Use Agent로, 사용자가 자연어로 지시하면 AI가 직접 브라우저를 띄워 클릭·입력·양식 제출까지 수행하는 "실행자" 역할을 맡는다. 동작 원리는 간단해서, 화면을 이미지로 캡처해 GPT-4o가 분석하고 다음 행동을 결정하는 루프를 반복한다. 기존 RPA가 좌표·셀렉터에 깨지기 쉬운 것과 달리, Operator는 화면 변화에 시각적으로 적응하고 별도 스크립트가 필요 없다는 점에서 비개발자에게도 접근성이 높다. 다만 비밀번호 직접 입력 금지, 결제 직전 사용자 승인 요구, 법적·금융 거래 위임 금지 같은 보안 원칙을 지켜야 하며, 현재는 사이트 호환성과 속도 측면에서 한계가 있어 점진적 확장이 예상된다.

## 인용 (key quotes, 짧게)

> "AI가 직접 웹 브라우저를 열고, 클릭하고, 입력하고, 양식을 제출하는 실행자 역할을 합니다."

> "화면을 이미지로 캡처하고 GPT-4o가 이를 분석하여 다음 행동을 결정하는 과정을 반복합니다."
