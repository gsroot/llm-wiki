## Chapter 12. Operator — AI가 직접 웹을 조작하다

지금까지의 ChatGPT는 정보를 제공하고 텍스트를 생성하는 "조언자" 역할이었습니다. 하지만 **Operator**는 한 단계 더 나아가 ChatGPT가 직접 웹 브라우저를 열고, 클릭하고, 입력하고, 양식을 제출하는 **실행자** 역할을 합니다. AI 에이전트의 시대가 본격적으로 시작된 것입니다.

## 

## Operator란 무엇인가?

**Operator**는 OpenAI가 개발한 Computer Use Agent입니다. 사용자의 지시를 받아 실제 웹 브라우저를 자율적으로 조작하여 원하는 작업을 완료합니다. "레스토랑 예약해줘", "항공권 검색해서 가장 저렴한 옵션 알려줘" 같은 지시를 내리면 Operator가 직접 해당 웹사이트에 접속하여 작업을 수행합니다.

기존 자동화 도구(RPA, 매크로)와의 차이점은 다음과 같습니다.

구분 | 기존 RPA/매크로 | Operator  
---|---|---  
설정 방식 | 클릭 경로 사전 프로그래밍 | 자연어 지시만으로 작동  
유연성 | 페이지 구조 변경 시 고장 | 화면을 이해하며 유연하게 대처  
필요 기술 | 자동화 도구 전문 지식 | 자연어 명령만 필요  
예외 처리 | 수동 개입 필요 | 스스로 대안 탐색  
  
## 

## Operator의 동작 원리

Operator는 **Computer Use Agent** 기술을 기반으로 합니다. 화면을 이미지로 캡처하고 GPT-4o가 이를 분석하여 다음 행동(클릭, 입력, 스크롤 등)을 결정하는 과정을 반복합니다.
    
    
    [동작 흐름]
    1. 사용자가 자연어로 작업 지시
             ↓
    2. Operator가 관련 웹사이트 접속
             ↓
    3. 화면 스크린샷 캡처 → GPT-4o가 분석
             ↓
    4. 다음 행동 결정 (클릭/입력/스크롤)
             ↓
    5. 결과 확인 → 3~4 반복
             ↓
    6. 작업 완료 또는 사용자 확인 요청
    

Operator는 작업 중 중요한 결정(결제, 최종 확인 등)이 필요한 시점에 사용자에게 **확인 요청** 을 보냅니다. 완전 자율이 아니라 사람이 핵심 단계를 검토하고 승인하는 **Human-in-the-Loop** 방식을 채택합니다.

## 

## 사용 가능 요금제와 접근 방법

현재 Operator는 **ChatGPT Pro** 요금제(월 $200)에서 먼저 출시되었으며, 점진적으로 다른 요금제로 확대되고 있습니다.

**접근 방법**
    
    
    ChatGPT 접속 → 좌측 사이드바 → "Operator" 선택
    또는: operator.chatgpt.com 직접 접속
    

tip

Operator는 2025년 초 미국에서 먼저 출시되어 점차 글로벌 지원을 확대하고 있습니다. 이용 가능 여부는 계정 설정에서 확인하거나 OpenAI 공식 홈페이지를 참조하세요.

## 

## 주요 활용 시나리오

### 시나리오 1: 온라인 예약 (레스토랑, 호텔, 항공)

가장 대표적인 Operator 활용 사례입니다. 조건을 말하면 Operator가 직접 예약 사이트에서 검색하고 예약까지 완료합니다.
    
    
    [레스토랑 예약 지시 예시]
    "다음 주 토요일 저녁 7시에 서울 강남구에서 
    이탈리안 레스토랑을 4인으로 예약해줘. 
    예산은 1인당 5만원 이하, 주차 가능한 곳이면 좋겠어."
    
    
    
    [항공권 검색 지시 예시]
    "5월 15일 서울→제주 왕복 항공권 검색해줘.
    편도 1시간 이내, 직항만, 
    가장 저렴한 3개 옵션 비교해서 알려줘."
    

### 시나리오 2: 쇼핑 가격 비교와 주문

여러 쇼핑몰을 직접 방문하여 가격을 비교하고 최적의 옵션을 찾아줍니다.
    
    
    [가격 비교 지시 예시]
    "삼성 갤럭시 S25 울트라 256GB 
    네이버쇼핑, 쿠팡, 11번가에서 가격 비교해서
    최저가와 배송 조건 알려줘."
    

### 시나리오 3: 웹 양식 작성과 제출

반복적인 온라인 신청서, 설문, 행정 양식 작성을 자동화합니다.
    
    
    [지시 예시]
    "구청 온라인 민원 포털에서 
    주민등록 열람 신청서를 작성하고 제출해줘.
    신청 목적은 '본인 확인용'이야."
    

### 시나리오 4: 반복적인 웹 기반 업무 자동화

매일 반복하는 웹 기반 업무를 Operator에게 위임합니다.
    
    
    [반복 업무 자동화 예시]
    - 매일 오전 경쟁사 3개사 블로그 새 글 확인 → 요약 보고
    - 주간 업계 뉴스 취합 → 이메일 초안 작성
    - 특정 제품 재고 상태 확인 → 재입고 시 알림
    - 채용 사이트에서 조건에 맞는 공고 수집
    

## 

## 보안 고려 사항

Operator는 강력하지만 그만큼 보안 주의가 필요합니다. 다음 원칙을 반드시 지켜야 합니다.

### 로그인 정보 취급

Operator는 사용자를 대신하여 웹사이트에 로그인할 수 있습니다. 이때 비밀번호를 직접 입력하는 것보다 **저장된 브라우저 자격증명** 을 활용하는 것이 안전합니다.
    
    
    [권장 방법]
    - 브라우저의 비밀번호 관리자에 로그인 정보 저장
    - Operator가 자동 완성을 통해 로그인하도록 설정
    - 민감한 계정의 비밀번호는 Operator에 직접 입력 금지
    

### 결제 관련 주의사항

Operator에게 결제가 포함된 작업을 맡길 때는 반드시 다음을 확인합니다.

주의 사항 | 권장 조치  
---|---  
결제 전 반드시 확인 요청하도록 설정 | "결제 직전에 나에게 확인 요청해줘" 명시  
한도 설정 | "10만원 이하만 진행해줘" 등 금액 제한  
신용카드 정보 직접 입력 금지 | 간편결제(카카오페이 등) 활용 권장  
  
### Operator에게 맡기면 안 되는 작업

아래 작업들은 Operator에게 위임하지 않는 것이 원칙입니다.

  * 법적 서류 서명 및 제출 (계약서, 소송 관련)
  * 대규모 금융 거래 (이체, 투자)
  * 타인 정보가 포함된 개인정보 처리
  * 보안 설정 변경 (계정 비밀번호 변경, 2FA 설정)
  * 회사 기밀 시스템 접근


tip

Operator 사용 중 언제든지 "멈춰"라고 말하면 작업을 즉시 중단합니다. 중요한 단계마다 "진행하기 전에 내게 알려줘"라는 지시를 추가하면 실수를 방지할 수 있습니다.

## 

## 현재 제약 사항과 향후 발전 방향

### 현재 제약 사항

  * **CAPTCHA** : 봇 방지 시스템에서 실패할 수 있음
  * **동적 웹사이트** : 복잡한 JavaScript 기반 사이트에서 불안정할 수 있음
  * **속도** : 사람보다 느릴 수 있으며 복잡한 작업은 수분 소요
  * **지역/언어** : 한국어 웹사이트 지원이 영어권보다 제한적일 수 있음
  * **요금제 제한** : 현재 Pro 요금제 중심, 월 사용량 제한 있음


### 향후 발전 방향

OpenAI는 Operator를 지속적으로 발전시키고 있습니다. 예상되는 발전 방향으로는 더 많은 요금제 지원 확대, 한국어 웹사이트 지원 강화, 멀티 탭 병렬 작업, 기업용 API 연동, 개인화된 웹 행동 패턴 학습 등이 있습니다.

## 

## 실습: Operator로 레스토랑 예약하기 시나리오

### 실습 시나리오 (사용법 시뮬레이션)

**목표** : Operator의 레스토랑 예약 흐름을 이해하고 올바른 지시 방법 학습

**참고** : 실제 예약은 Pro 계정이 필요하며, 아래는 사용법 시뮬레이션입니다.

### 단계 1: Operator 접속
    
    
    ChatGPT → 좌측 메뉴 → "Operator" 선택
    

### 단계 2: 효과적인 지시문 작성

불명확한 지시보다 조건이 구체적일수록 Operator가 더 정확하게 작동합니다.
    
    
    [나쁜 지시 — 너무 모호]
    "레스토랑 예약해줘"
    
    [좋은 지시 — 구체적 조건 포함]
    "네이버 예약 또는 캐치테이블에서 다음 조건으로 
    레스토랑을 검색하고 예약해줘:
    - 날짜: 2026년 4월 18일 (토요일)
    - 시간: 저녁 7시
    - 인원: 2명
    - 지역: 서울 종로구 또는 중구
    - 음식: 일식 (오마카세 또는 스시)
    - 예산: 1인당 10만원 내외
    - 기타: 창가 자리 선호, 주류 반입 가능 여부 확인
    
    예약 전에 선택한 레스토랑 정보(메뉴, 리뷰, 위치)를 
    먼저 나에게 보여주고 확인을 받아줘."
    

### 단계 3: Operator 작업 모니터링

Operator가 작업을 시작하면 화면에서 실시간으로 진행 상황을 확인할 수 있습니다.
    
    
    [Operator 작업 흐름 — 예상 화면]
    1. 캐치테이블 접속
    2. 조건 입력 (날짜, 인원, 지역)
    3. 검색 결과 목록 확인
    4. 조건에 맞는 레스토랑 3개 선별
       → 사용자에게 확인 요청: "다음 중 어떤 곳으로 예약할까요?"
    5. 사용자 선택 후
       → 예약 페이지 접속
       → 개인 정보 입력 (사전 저장된 정보 활용)
       → 결제/예약 확정 전 최종 확인 요청
    6. 사용자 승인 후 예약 완료
       → 예약 확인서 캡처하여 전달
    

### 단계 4: 확인 및 완료

Operator가 최종 확인을 요청하면 예약 내용을 꼼꼼히 검토한 뒤 승인합니다.
    
    
    [Operator 최종 확인 메시지 예시]
    "아래 내용으로 예약을 진행할까요?
    
    레스토랑: 스시 료 (서울 종로구)
    날짜/시간: 4월 18일 토요일 오후 7시
    인원: 2명
    코스: 저녁 오마카세 (1인 98,000원)
    총 금액: 196,000원
    
    [확인하고 예약 진행] [취소]"
    

tip

Operator를 처음 사용할 때는 저위험 작업(가격 비교, 정보 수집 등)부터 시작하세요. 실제 예약이나 결제가 포함된 작업은 몇 번의 연습 후 신뢰가 쌓였을 때 진행하는 것이 안전합니다.

## 

## Part 4 마무리

Part 4에서는 ChatGPT를 단순한 텍스트 생성 도구에서 진정한 업무 파트너로 만드는 네 가지 고급 기능을 살펴봤습니다.

  * **Custom GPTs** : 목적에 맞게 특화된 AI 어시스턴트를 직접 설계
  * **Memory와 Custom Instructions** : AI가 나의 맥락을 기억하고 맞춤형으로 응답
  * **멀티모달 기능** : 음성·이미지·파일을 넘나드는 통합 작업 환경
  * **Operator** : AI가 직접 행동하는 에이전트 시대의 개막


이 기능들을 조합하면 ChatGPT는 단순 질의응답 도구를 넘어 진정한 의미의 AI 어시스턴트로 진화합니다. 다음 Part 5에서는 API와 Codex를 활용한 개발자를 위한 고급 활용법을 다룹니다.
