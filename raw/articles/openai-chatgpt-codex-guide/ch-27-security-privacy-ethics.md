## Chapter 27. 보안·개인정보·윤리

AI 도구를 업무에 활용할 때 가장 먼저 부딪히는 현실적인 우려는 "내가 입력한 데이터는 어디로 가는가?"입니다. 그리고 AI가 생성한 결과물을 그대로 사용해도 되는지, 혹시 사실과 다른 내용이 포함된 건 아닌지 걱정도 됩니다. 이 챕터에서는 이러한 보안·프라이버시·윤리 문제를 체계적으로 다루고, 조직에서 활용할 수 있는 실용적인 가이드라인을 제공합니다.

## 데이터 프라이버시 정책 이해

### ChatGPT의 데이터 처리 방식

ChatGPT에 입력하는 내용이 어떻게 처리되는지 정확히 알아야 안심하고 사용할 수 있습니다.

**요금제별 데이터 정책 비교:**

항목 | Free / Plus | Team | Enterprise  
---|---|---|---  
대화 내용 학습 활용 | 기본 활용 (설정에서 끄기 가능) | 기본 제외 | 기본 제외  
데이터 보존 기간 | 30일 | 설정 가능 | 설정 가능  
DPA 제공 | 미제공 | 제공 | 제공  
  
### 학습 데이터 제외 옵션

Free/Plus 사용자라면 직접 설정을 변경해야 합니다.
    
    
    학습 데이터 제외 설정 경로:
    Settings → Data controls → Improve the model for everyone → OFF
    

이 설정을 끄면 이후 대화 내용은 모델 학습에 사용되지 않습니다. 단, 이미 입력된 과거 대화는 소급 적용되지 않습니다.

**Team과 Enterprise 요금제는 이 설정이 기본적으로 비활성화되어 있어 별도 조치 없이도 학습에 사용되지 않습니다.**

### API 사용 시 데이터 정책

OpenAI API를 통해 ChatGPT를 사용하는 경우에는 더 명확한 보호가 제공됩니다.

  * API로 전송된 데이터는 기본적으로 모델 학습에 사용되지 않음
  * 30일 후 자동 삭제 (기본값)
  * Zero Data Retention(ZDR) 옵션으로 즉시 삭제 요청 가능 (Enterprise 계약 시)


### Data Processing Addendum (DPA)

DPA는 OpenAI와 기업 간의 데이터 처리 계약서입니다. GDPR, 한국 개인정보보호법 등 각국 법규 준수를 위해 반드시 검토해야 합니다.

  * Team/Enterprise 고객은 DPA 자동 적용
  * 개인정보 처리 목적, 보존 기간, 삭제 절차 등이 명시됨
  * 법무팀/DPO(개인정보보호책임자)와 함께 검토 권장

## 민감 정보 취급 가이드라인

### 입력하면 안 되는 정보

아무리 보안이 강화된 요금제를 사용하더라도, 다음 정보는 절대 AI에 입력해서는 안 됩니다.

**절대 입력 금지:**

  * 비밀번호, API 키, 액세스 토큰
  * 주민등록번호, 여권번호 등 고유 식별정보
  * 신용카드·계좌번호 등 금융정보
  * 고객의 개인정보 (이름, 연락처, 주소 등)
  * 비밀 유지 의무가 있는 계약서 내용
  * 미공개 재무정보, 내부 영업 전략


tip

"이 정보를 외부인에게 보여줄 수 있는가?"라는 기준을 적용하세요. 그 답이 "아니오"라면 AI에도 입력하지 마세요.

### 가명처리·마스킹 기법

민감 정보가 포함된 문서를 AI로 처리해야 할 때는 반드시 비식별화 처리를 먼저 수행하세요.

**실전 마스킹 예시:**
    
    
    원본:
    "홍길동(800101-1234567) 고객이 2024년 3월 5일 
     서울시 강남구 테헤란로 123에서 상담을 요청했습니다."
    
    마스킹 처리 후:
    "[고객명](생년월일-[주민번호]) 고객이 [날짜] 
     [지역]에서 상담을 요청했습니다."
    

**자동화 마스킹 도구:**
    
    
    import re
    
    def mask_sensitive_info(text):
        # 주민등록번호 마스킹
        text = re.sub(r'\d{6}-\d{7}', '[주민번호]', text)
        # 전화번호 마스킹
        text = re.sub(r'01[0-9]-\d{3,4}-\d{4}', '[전화번호]', text)
        # 이메일 마스킹
        text = re.sub(r'[\w.]+@[\w.]+', '[이메일]', text)
        # 신용카드 마스킹
        text = re.sub(r'\d{4}-\d{4}-\d{4}-\d{4}', '[카드번호]', text)
        return text
    
    # 사용 예시
    original = "홍길동(010-1234-5678)의 주민번호는 800101-1234567입니다."
    masked = mask_sensitive_info(original)
    print(masked)
    # 출력: 홍길동([전화번호])의 주민번호는 [주민번호]입니다.
    

### 데이터 분류 체계

조직 내 데이터를 명확하게 분류하면 AI 활용 가능 여부를 신속하게 판단할 수 있습니다.

등급 | 정의 | AI 활용 가능 여부  
---|---|---  
공개(Public) | 이미 공개된 정보 | 자유롭게 활용 가능  
내부(Internal) | 사내 공유 가능 정보 | Team/Enterprise에서 활용 가능  
비밀(Confidential) | 제한된 인원만 접근 | 마스킹 후 신중하게 활용  
극비(Restricted) | 최고 기밀 | AI 입력 금지

## AI 생성 콘텐츠의 저작권과 책임

### 현재 법적 프레임워크

AI가 생성한 콘텐츠의 저작권은 전 세계적으로 아직 정립 중인 분야입니다.

**주요 원칙:**

  * 미국, 한국 등 대부분의 국가에서 **AI 단독 창작물은 저작권 보호를 받지 못함**
  * 인간이 AI를 도구로 활용하여 창작한 결과물은 인간 저작자의 저작권으로 인정될 수 있음
  * 단, 창작적 기여도에 대한 논쟁이 지속 중


**OpenAI의 이용약관에 따른 소유권:**

OpenAI는 사용자가 ChatGPT를 통해 생성한 출력물에 대한 권리를 사용자에게 귀속시키고 있습니다. 다만 이용약관 준수를 전제로 합니다.

### 기업 내 AI 저작물 관리 정책

기업은 다음 정책을 사전에 수립해야 합니다.

  1. **명시적 표기** : AI 생성 콘텐츠임을 내부 문서에 표기하는 기준 수립
  2. **검토 의무** : 외부 발행 콘텐츠는 반드시 인간 편집자의 검토와 승인 필요
  3. **저작물 등록** : 중요 창작물은 인간의 창작적 기여가 명확한 경우에만 저작권 등록 고려
  4. **책임 소재 명확화** : AI 생성 오류로 인한 피해 발생 시 책임 소재 사전 정의


### 표절·유사성 검토

AI가 기존 저작물을 그대로 재현하거나 과도하게 유사한 내용을 생성할 수 있습니다.

**검토 도구 활용:**

  * **Copyleaks** : AI 생성 콘텐츠 감지 및 표절 검사
  * **Turnitin** : 학술 영역에서 주로 사용
  * **iThenticate** : 기업·출판 영역


외부 발행 콘텐츠는 반드시 유사성 검토 후 발행하는 워크플로우를 구축하세요.

## 환각(Hallucination) 대응 전략

### 환각의 원인과 유형

AI 환각이란 모델이 사실처럼 들리지만 실제로는 잘못된 정보를 생성하는 현상입니다.

**환각의 주요 유형:**

유형 | 예시  
---|---  
사실 오류 | 존재하지 않는 논문, 법조문, 통계 인용  
날짜/수치 오류 | 틀린 날짜, 부정확한 수치  
인물 정보 오류 | 실존 인물의 경력, 발언 왜곡  
URL/참조 오류 | 존재하지 않는 URL 제시  
  
환각은 **확신에 찬 어조로 표현되기 때문에 구별이 어렵습니다**. 이것이 팩트 체크가 반드시 필요한 이유입니다.

### 팩트 체크 프로세스 설계

**3단계 팩트 체크 프로세스:**
    
    
    Step 1. 비판적 읽기
      - AI 출력물의 모든 수치, 날짜, 이름, 법조문 목록 작성
      - "이것이 사실인가?"를 스스로 질문
    
    Step 2. 교차 검증
      - 신뢰할 수 있는 1차 출처로 확인
        (공식 기관 사이트, 학술 데이터베이스, 원문 문헌)
      - 다른 AI 모델로 교차 확인도 보완 수단이 됨
    
    Step 3. 출처 표기
      - 검증된 정보에는 출처 명시
      - 검증 불가 정보는 삭제 또는 "확인 필요" 표기
    

### 환각 방지 프롬프트 기법
    
    
    # 환각 억제 프롬프트 예시
    
    프롬프트:
    "다음 질문에 답변할 때 반드시 지켜야 할 규칙:
    1. 확실하지 않은 정보는 '확인이 필요합니다'라고 명시하세요.
    2. 통계나 수치를 언급할 때는 출처를 함께 제시하세요.
    3. 출처를 모른다면 추측하지 말고 솔직히 말하세요.
    4. URL은 제공하지 마세요 (존재하지 않을 수 있습니다).
    
    질문: [여기에 질문 입력]"

## AI 윤리 가이드라인

### 편향(Bias) 인식과 대응

AI 모델은 학습 데이터에 내재된 편향을 반영할 수 있습니다.

**주요 편향 유형:**

  * **성별 편향** : 특정 직업이나 역할을 특정 성별과 연결
  * **문화/지역 편향** : 서구 중심적 관점이나 특정 문화권의 가치관 반영
  * **확증 편향** : 사용자의 기존 의견을 지지하는 방향으로 편향된 응답


**대응 방법:**

  1. 중요한 의사결정에 AI 단독으로 의존하지 않기
  2. 다양한 관점에서 프롬프트를 재구성해 테스트
  3. 출력 결과의 다양성과 포용성을 인간이 검토


### 투명성과 설명 가능성

AI를 활용하여 만든 결과물을 사용할 때는 투명성이 중요합니다.

  * **내부 문서** : AI 보조로 작성되었음을 표기 (예: "AI 초안 기반 작성")
  * **고객 대면 콘텐츠** : AI 활용 여부에 대한 공시 정책 수립
  * **의사결정 보조** : AI 권고사항을 그대로 따르지 않고 근거를 설명할 수 있어야 함


### 인간 감독(Human-in-the-Loop) 원칙

**AI는 보조 도구이며, 최종 판단과 책임은 반드시 인간에게 있습니다.**

**적용 원칙:**

  * 고위험 의사결정(인사, 의료, 법적 판단 등)에는 AI 자동화 금지
  * AI 출력물의 최종 승인 권한을 반드시 인간이 보유
  * 주기적인 AI 활용 결과 감사(Audit) 수행

## 조직용 AI 사용 정책(AUP) 템플릿

아래는 조직에서 즉시 활용할 수 있는 AI 사용 정책(Acceptable Use Policy) 템플릿입니다. 조직 상황에 맞게 수정하여 사용하세요.
    
    
    # [조직명] AI 도구 사용 정책 (AUP)
    버전: 1.0 | 시행일: 20XX년 XX월 XX일
    
    ## 1. 목적
    본 정책은 [조직명] 구성원이 ChatGPT 등 AI 도구를 안전하고 윤리적으로 
    활용하기 위한 기준을 정의합니다.
    
    ## 2. 허용되는 사용
    - 문서 초안 작성 및 편집 보조
    - 아이디어 브레인스토밍 및 리서치 보조
    - 코드 작성 및 검토 보조
    - 번역 및 요약 작업
    - 교육 및 학습 목적
    
    ## 3. 금지되는 사용
    - 고객 개인정보 및 민감 정보 입력
    - 비공개 재무 정보 및 영업 기밀 입력
    - 비밀번호, API 키, 인증 정보 입력
    - AI 생성 콘텐츠를 검토 없이 외부 발행
    - 불법적이거나 차별적인 콘텐츠 생성
    
    ## 4. 데이터 취급 기준
    - 회사 데이터는 [승인된 요금제명]에서만 사용
    - 극비(Restricted) 등급 데이터는 AI 입력 금지
    - 비밀(Confidential) 등급 데이터는 마스킹 후 사용
    
    ## 5. 결과물 검토 의무
    - AI 생성 결과물은 반드시 담당자가 검토 후 사용
    - 외부 발행 콘텐츠는 추가 팩트 체크 필수
    - 수치, 법조문, 통계는 반드시 원출처 확인
    
    ## 6. 책임 및 위반 처리
    - 본 정책 위반 시 [인사 규정에 따른 처분]
    - 보안 사고 발생 시 즉시 [담당 부서]에 신고
    
    ## 7. 정책 검토
    본 정책은 연 1회 이상 검토하여 최신 상황을 반영합니다.
    

tip

AUP는 구성원이 실제로 읽고 이해할 수 있도록 간결하게 작성하는 것이 중요합니다. 법률 문서처럼 복잡하게 만들면 아무도 읽지 않습니다. 핵심 내용을 1페이지로 요약한 "AI 사용 카드"를 함께 제공하는 것도 효과적입니다.
