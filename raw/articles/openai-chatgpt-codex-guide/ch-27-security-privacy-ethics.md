- 데이터 프라이버시 정책 이해
  - ChatGPT의 데이터 처리 방식
  - 학습 데이터 제외 옵션
  - API 사용 시 데이터 정책
  - Data Processing Addendum (DPA)
- 민감 정보 취급 가이드라인
  - 입력하면 안 되는 정보
  - 가명처리·마스킹 기법
  - 데이터 분류 체계
- AI 생성 콘텐츠의 저작권과 책임
  - 현재 법적 프레임워크
  - 기업 내 AI 저작물 관리 정책
  - 표절·유사성 검토
- 환각(Hallucination) 대응 전략
  - 환각의 원인과 유형
  - 팩트 체크 프로세스 설계

  - 인간 감독(Human-in-the-Loop) 원칙
- 조직용 AI 사용 정책(AUP) 템플릿

## 핵심 요약 (자가 작문)

업무에 AI 도구를 들이려면 데이터 보안·개인정보·윤리 세 축에서 사전 준비가 필요하다. 이 장은 요금제별로 학습 데이터 사용 여부와 보존 기간이 어떻게 다른지 설명하고, DPA 같은 법적 장치와 SCIM·SSO 같은 운영 통제 수단을 함께 다룬다. 민감 정보는 외부 공개 가능성을 기준으로 분류하고 가명처리·마스킹으로 입력 위험을 낮추며, 생성물에 대한 저작권과 책임은 조직 내부 정책으로 명문화한다. 환각은 원인 유형별로 팩트 체크 프로세스와 프롬프트 기법으로 완화하고, 편향 인식·설명 가능성·인간 감독 원칙을 윤리 베이스라인으로 삼는다. 마지막으로 즉시 도입 가능한 AUP(AI 사용 정책) 템플릿을 제공해 조직이 빠르게 정합성을 확보할 수 있도록 돕는다.
