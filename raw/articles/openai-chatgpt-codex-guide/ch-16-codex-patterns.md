## 핵심 요약 (자가 작문)

16장은 앞 장들에서 익힌 기본기를 실무 시나리오에 응용하는 5가지 패턴을 제시한다. 첫째, 레거시 마이그레이션은 React Class → Hooks 변환이나 언어 버전 업그레이드처럼 디렉터리 단위로 쪼개 점진적으로 진행해야 안정적이다. 둘째, 문서화 자동 생성은 README, OpenAPI/Swagger 스펙, 코드 주석/Docstring까지 포괄해 코드와 문서의 동기화 부담을 낮춘다. 셋째, 보안 스캔에서는 의존성 취약점 업데이트와 코드 내 결함 탐지가 가능하지만 자동 수정 PR을 그대로 머지하지 말고 시니어·보안 전문가가 반드시 리뷰해야 한다. 넷째, 대규모 코드베이스에는 잘 짠 AGENTS.md와 태스크 분할·병렬 실행 전략이 필수다. 마지막으로 비개발자도 데이터 분석 스크립트나 자동화 도구 제작에 Codex를 활용할 수 있어, 사용 범위가 개발팀 너머로 확장된다.
