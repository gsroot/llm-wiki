# Chapter 04. 효과적인 프롬프트 작성법 — AI와 잘 대화하는 기술

## 프롬프트란 무엇인가

프롬프트(Prompt)는 AI에게 전달하는 입력 텍스트입니다. 같은 AI 모델이라도 어떤 프롬프트를 입력하느냐에 따라 결과물의 품질이 극적으로 달라집니다. 단순히 "요약해줘"와 "마케팅 경험이 없는 스타트업 CEO에게 보내는 투자자 보고서 형식으로, 핵심 지표 3가지를 중심으로 200자 이내로 요약해줘"의 결과는 전혀 다릅니다.

프롬프트 작성을 잘 하는 것은 AI를 잘 활용하는 것의 절반입니다. 좋은 프롬프트는 AI가 당신이 원하는 방향을 정확히 이해하고 최적의 답변을 제공하도록 안내합니다.

---

## 프롬프트의 4가지 핵심 구성 요소

효과적인 프롬프트는 네 가지 요소로 구성됩니다. 이를 RCIF 프레임워크라고 부릅니다.

### Role (역할)

AI에게 어떤 전문가 또는 페르소나로 답변해야 하는지 지정합니다. 역할을 설정하면 AI는 해당 분야의 지식과 어조로 답변합니다.

```
당신은 10년 경력의 카피라이터입니다.
당신은 고등학교 영어 선생님입니다.
당신은 스타트업 투자 전문 VC 파트너입니다.
```

### Context (맥락)

배경 정보와 상황을 제공합니다. AI가 적절한 답변을 위해 알아야 할 정보를 모두 포함합니다.

```
우리 회사는 B2B SaaS 스타트업으로, 
주요 고객은 50인 이하 중소기업 HR 담당자입니다.
현재 월 구독자는 300명이며, 이탈률이 15%로 높은 상황입니다.
```

### Instruction (지시)

정확히 무엇을 해야 하는지 구체적으로 명시합니다. 모호한 지시보다 구체적인 지시가 훨씬 좋은 결과를 냅니다.

```
[모호] 이메일 써줘
[구체] 고객 이탈 방지를 위한 재활성화 이메일을 작성해주세요.
       제목 포함, 본문 200자 이내, 혜택 1가지 제시 포함
```

### Format (형식)

출력 형식을 지정합니다. 표, 목록, JSON, 마크다운 등 원하는 형식을 명시하면 바로 활용 가능한 결과물을 얻을 수 있습니다.

```
결과를 표 형식으로 정리해주세요. (3열: 항목, 설명, 예시)
마크다운 불릿 리스트로 5가지 핵심만 정리해주세요.
JSON 형식으로 출력해주세요.
```

## Few-shot 프롬프팅: 예시를 보여주는 기법

Few-shot 프롬프팅은 원하는 출력의 형태를 예시로 보여주는 방법입니다. AI는 제시된 패턴을 학습하여 동일한 형식으로 새로운 내용을 생성합니다.

```
아래 형식으로 제품 소개 문구를 3개 더 작성해주세요.

[예시 1]
제품: 스마트 텀블러
문구: 온도를 기억하는 텀블러. 당신의 하루를 더 따뜻하게.

[예시 2]
제품: 무선 충전 패드
문구: 선 없이도 연결되는 일상. 내려놓는 순간 충전이 시작됩니다.

이제 다음 제품들에 대해 작성해주세요:
- 스마트 체중계
- 공기청정기
- 스탠딩 데스크
```

Few-shot 기법은 특히 다음 상황에서 효과적입니다.

* 특정 문체나 어조를 유지해야 할 때
* 브랜드 가이드라인에 맞는 콘텐츠를 만들 때
* 데이터 변환 형식이 복잡할 때

## Chain-of-Thought: 단계별 사고를 유도하는 기법

Chain-of-Thought(CoT, 연쇄 추론) 프롬프팅은 AI가 결론을 바로 내리지 않고, 단계적으로 사고 과정을 거치도록 유도하는 기법입니다. 복잡한 분석이나 문제 해결에 특히 효과적입니다.

**CoT 없는 프롬프트:**

```
이 사업 계획서의 실패 가능성을 분석해줘.
```

**CoT 적용 프롬프트:**

```
이 사업 계획서의 실패 가능성을 분석해주세요.
다음 단계로 진행해주세요:

1단계: 시장 분석 — 시장 크기와 경쟁 환경을 평가합니다
2단계: 재무 검토 — 수익 모델과 비용 구조의 현실성을 검토합니다
3단계: 실행 가능성 — 팀 역량과 리소스의 적합성을 평가합니다
4단계: 리스크 요인 — 주요 위험 요소 3가지를 도출합니다
5단계: 종합 결론 — 위 분석을 바탕으로 전반적인 평가를 제시합니다
```

단순히 "단계별로 생각해줘" 또는 "Let's think step by step"이라는 한 문장만 추가해도 AI의 답변 품질이 크게 향상됩니다. 특히 o3 모델에서는 CoT가 내부적으로 자동으로 적용됩니다.

## 시스템 프롬프트와 Custom Instructions 활용

### 시스템 프롬프트란

시스템 프롬프트는 대화 전체에 걸쳐 AI의 행동을 제어하는 배경 지침입니다. API를 직접 사용하는 개발자는 `system` 역할로 지정하지만, ChatGPT 웹 사용자는 Custom Instructions를 통해 유사한 효과를 낼 수 있습니다.

### Custom Instructions 고급 활용

단순한 직업 정보 입력을 넘어, Custom Instructions를 전략적으로 작성하면 매우 강력한 효과를 냅니다.

```
[나에 대한 정보]
- 스타트업 CTO, 백엔드 개발 10년 경력 (Python/Node.js 전문)
- 현재 MSA(마이크로서비스 아키텍처)로 서비스 전환 중
- 팀: 개발자 5명, 데브옵스 1명
- 인프라: AWS (ECS, RDS, Redis, SQS 사용)

[응답 방식]
- 기술적 설명 시 실제 코드 예시를 반드시 포함해주세요
- AWS 서비스 중심으로 솔루션을 제안해주세요
- 보안 고려사항은 별도 섹션으로 분리해주세요
- 트레이드오프(장단점)를 명확히 설명해주세요
```

## 나쁜 프롬프트 vs 좋은 프롬프트: Before/After 비교

### 사례 1: 번역 요청

| 구분     | 프롬프트                                                                         |
| ------ | ---------------------------------------------------------------------------- |
| Before | 이 글 번역해줘                                                                     |
| After  | 다음 한국어 문단을 비즈니스 영어로 번역해주세요. 격식체(formal)를 유지하고, 미국 독자를 대상으로 자연스러운 표현을 사용해주세요. |

After 프롬프트는 언어, 어조, 대상 독자를 명시하여 바로 사용 가능한 번역 결과를 얻을 수 있습니다.

### 사례 2: 아이디어 요청

| 구분     | 프롬프트                                                                                                           |
| ------ | -------------------------------------------------------------------------------------------------------------- |
| Before | 마케팅 아이디어 줘                                                                                                     |
| After  | 월 예산 50만원 이내, 인스타그램과 유튜브 쇼츠 채널을 운영 중인 반려동물 용품 쇼핑몰을 위한 콘텐츠 마케팅 아이디어를 5가지 제안해주세요. 각 아이디어마다 예상 비용과 예상 효과를 포함해주세요. |

### 사례 3: 코드 요청

| 구분     | 프롬프트                                                                                                                                     |
| ------ | ---------------------------------------------------------------------------------------------------------------------------------------- |
| Before | 파이썬 코드 짜줘                                                                                                                                |
| After  | Python 3.11을 사용하여, 폴더 내 모든 .jpg 파일의 이름을 'YYYYMMDD\_순번.jpg' 형식으로 일괄 변경하는 스크립트를 작성해주세요. 파일의 생성일 기준으로 정렬하고, 이미 같은 이름이 존재하면 덮어쓰지 않도록 처리해주세요. |

### 사례 4: 분석 요청

| 구분     | 프롬프트                                                                                                                                                |
| ------ | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| Before | 이 데이터 분석해줘                                                                                                                                          |
| After  | 첨부한 엑셀 파일은 우리 쇼핑몰의 최근 3개월 주문 데이터입니다. 다음 세 가지를 분석해주세요: (1) 요일별 주문량 패턴, (2) 상품 카테고리별 매출 비중, (3) 재구매율이 가장 높은 고객 연령대. 각 분석 결과를 표와 핵심 인사이트 1문장으로 정리해주세요. |

## GPT-4o vs o3: 프롬프트 전략의 차이

모델 특성에 따라 최적의 프롬프트 전략도 달라집니다.

| 항목      | GPT-4o                | o3                       |
| ------- | --------------------- | ------------------------ |
| 응답 속도   | 빠름                    | 느림 (심층 추론 시)             |
| 적합한 작업  | 창작, 요약, 일상 대화, 이미지 분석 | 수학, 코딩, 복잡한 논리, 다단계 추론   |
| 프롬프트 전략 | 명확하고 간결하게, 예시 포함      | 목표와 제약 조건을 명확히, 추론 단계 요청 |
| CoT 필요성 | 수동 지정 효과적             | 내부적으로 자동 적용              |

**GPT-4o용 프롬프트 예시** (창작 작업):

```
당신은 유머러스한 IT 칼럼니스트입니다.
"스마트홈 기기들이 반란을 일으킨다면" 이라는 주제로
300자 내외의 짧은 풍자 에세이를 써주세요.
일상적인 상황과 과장된 상상을 섞어 독자가 웃을 수 있도록 해주세요.
```

**o3용 프롬프트 예시** (복잡한 분석):

```
다음 조건을 모두 만족하는 데이터베이스 인덱싱 전략을 설계해주세요:
- PostgreSQL 환경, 테이블 행 수: 5천만 건
- 쿼리 패턴: 날짜 범위 + 사용자 ID 복합 조건 검색 (90%), 전체 텍스트 검색 (10%)
- 쓰기 작업 비율: 읽기:쓰기 = 7:3
- 목표: P95 응답시간 100ms 이하

각 전략의 장단점, 예상 성능 향상, 구체적인 SQL 인덱스 생성 쿼리까지 포함해주세요.
```

## [실습] 업무 이메일 프롬프트를 단계적으로 개선하기

**시나리오:** 거래처에 납기 지연을 알리는 이메일을 AI로 작성하면서, 프롬프트를 단계적으로 개선하여 결과물의 품질이 어떻게 달라지는지 직접 체험합니다.

**단계:**

1. **Level 1 - 기본 프롬프트로 시작합니다:**

```
납기 지연 이메일 써줘
```

AI의 첫 번째 응답을 저장합니다.

2. **Level 2 - 역할과 맥락을 추가합니다:**

```
당신은 제조업체의 영업 담당자입니다.
주문 번호 ORD-2026-0401에 대한 납기가 
예정일(4월 15일)보다 1주일 지연될 예정입니다.
지연 사유는 부품 수급 문제입니다.
거래처 담당자(김철수 부장)에게 사과 이메일을 써주세요.
```

3. **Level 3 - 형식과 구체적 요구사항을 추가합니다:**

```
당신은 제조업체의 영업 담당자입니다.
주문 번호 ORD-2026-0401에 대한 납기가 
예정일(4월 15일)보다 1주일 지연될 예정입니다.
지연 사유는 부품 수급 문제입니다.
거래처 담당자(김철수 부장)에게 사과 이메일을 써주세요.

다음 요소를 반드시 포함하세요:
- 진심 어린 사과와 구체적인 새 납기일 (4월 22일) 명시
- 지연으로 인한 불편에 대한 보상안 (다음 주문 5% 할인)
- 진행 상황을 주 1회 업데이트하겠다는 약속
- 이메일 형식: 제목 포함, 본문 300자 이내, 격식체 유지
```

**실행 결과:**

Level 1의 결과물은 일반적이고 사용하기 어려운 이메일이 나옵니다. Level 2에서는 상황에 맞는 이메일이 생성되지만 바로 보내기에는 부족합니다. Level 3에서는 제목, 사과, 보상안, 재납기일이 포함된 바로 발송 가능한 이메일이 완성됩니다.

세 단계의 결과물을 비교하면서, 프롬프트에 얼마나 구체적인 정보를 담느냐가 결과물 품질을 결정한다는 사실을 직접 확인할 수 있습니다.

**프롬프트 개선의 황금 법칙**

좋은 프롬프트를 처음부터 완벽하게 작성하려 하지 마세요. 기본 프롬프트로 시작하여 결과를 보고, 부족한 부분을 추가해 나가는 반복적 개선(Iterative Refinement) 방식이 실무에서 가장 효과적입니다. AI와의 대화는 한 번으로 끝내려 하지 말고, 대화를 이어가며 점진적으로 원하는 결과물에 도달하세요.
