## A. 자주 쓰는 프롬프트 템플릿 모음

아래 20가지 프롬프트는 복사 후 **[괄호] 안의 내용만 바꿔서** 바로 사용할 수 있도록 설계되었습니다. 업무 유형별로 분류하였으니 필요한 항목을 찾아 활용하세요.

* * *

## 문서 작성

### 1. [문서] 업무 보고서 초안 작성

**용도:** 회의 결과나 진행 상황을 상사에게 보고하는 문서 초안 작성

**프롬프트:**

> 아래 내용을 바탕으로 **업무 보고서** 초안을 작성해줘.
> 
>   * 보고 목적: [보고 목적]
>   * 주요 내용: [핵심 내용 3~5개]
>   * 현재 상태: [진행 현황]
>   * 다음 액션: [후속 조치 계획]
> 

> 
> 문체는 격식체(~습니다)로, A4 1장 분량으로 작성해줘.

**활용 팁:** "위 보고서를 임원 보고용으로 요약해줘"처럼 후속 프롬프트로 버전을 다양화할 수 있습니다.

### 2. [문서] 비즈니스 이메일 작성

**용도:** 외부 파트너·고객에게 보내는 공식 이메일 초안 생성

**프롬프트:**

> 다음 상황에 맞는 비즈니스 이메일을 써줘.
> 
>   * 수신자: [수신자 직책/이름]
>   * 이메일 목적: [미팅 요청 / 견적 문의 / 계약 확인 등]
>   * 핵심 전달 내용: [내용 요약]
>   * 요청 사항: [상대방에게 원하는 액션]
>   * 톤: [정중하고 간결하게 / 친근하게 / 등]
> 

> 
> 제목 포함해서 작성해줘.

**활용 팁:** 영어 이메일이 필요하면 마지막에 "영어로 작성해줘"를 추가하세요.

### 3. [문서] 기획서 목차 + 개요 작성

**용도:** 신규 프로젝트·서비스·캠페인 기획서의 뼈대 잡기

**프롬프트:**

> [프로젝트/서비스명]에 대한 기획서 목차와 각 섹션 개요를 작성해줘.
> 
>   * 프로젝트 개요: [한 줄 설명]
>   * 목표 독자: [내부 경영진 / 투자자 / 고객 등]
>   * 반드시 포함할 내용: [예산, 일정, KPI 등]
> 

> 
> 목차는 5~7개 섹션으로 구성하고 각 섹션에 2~3줄 설명을 붙여줘.

**활용 팁:** 목차가 나오면 "3번 섹션을 A4 1장 분량으로 상세 작성해줘"처럼 섹션별로 이어서 작성합니다.

### 4. [문서] 회의록 정리

**용도:** 회의 메모나 녹취 내용을 구조화된 회의록으로 정리

**프롬프트:**

> 아래 회의 메모를 공식 회의록 형식으로 정리해줘.
> 
> [회의 메모 붙여넣기]
> 
> 형식은 다음과 같이 해줘: 1. 일시 / 참석자 2. 논의 안건 3. 결정 사항 4. 액션 아이템 (담당자, 기한 포함)

**활용 팁:** 음성 녹취 텍스트를 붙여넣으면 불필요한 발화를 제거하고 요점만 추출합니다.

## 데이터 분석

### 5. [분석] 데이터 요약 및 인사이트 추출

**용도:** CSV/표 형태의 데이터에서 핵심 인사이트 도출

**프롬프트:**

> 아래 데이터를 분석해서 주요 인사이트 5가지를 불릿 포인트로 정리해줘.
> 
> [데이터 또는 표 붙여넣기]
> 
>   * 분석 목적: [매출 증가 원인 파악 / 이탈 고객 패턴 분석 등]
>   * 특히 주목할 지표: [특정 컬럼명 또는 지표]
> 

> 
> 각 인사이트에는 데이터 근거(수치)를 포함해줘.

**활용 팁:** 데이터가 크면 먼저 집계 통계(평균, 합계 등)를 붙여넣고 원본 데이터는 샘플 20행만 제공하세요.

### 6. [분석] 시각화 코드 생성

**용도:** 데이터 시각화용 Python 코드 즉시 생성

**프롬프트:**

> 아래 데이터를 시각화하는 Python 코드를 작성해줘.
> 
>   * 데이터 구조: [컬럼명과 샘플 데이터]
>   * 시각화 유형: [막대 그래프 / 라인 차트 / 히트맵 등]
>   * 라이브러리: [matplotlib / seaborn / plotly 중 선택]
>   * 강조할 요소: [특정 기간 / 특이값 등]
> 

> 
> 코드에 한글 주석을 달아줘.

**활용 팁:** plotly를 선택하면 인터랙티브 차트가 생성되어 대시보드에 활용하기 좋습니다.

### 7. [분석] A/B 테스트 결과 해석

**용도:** A/B 테스트 결과 데이터를 통계적으로 해석하고 결론 도출

**프롬프트:**

> 아래 A/B 테스트 결과를 분석하고 의사결정 권고안을 작성해줘.
> 
>   * 테스트 목적: [버튼 색상 변경이 클릭률에 미치는 영향 등]
>   * 그룹 A 결과: [샘플 수, 전환율 등]
>   * 그룹 B 결과: [샘플 수, 전환율 등]
>   * 테스트 기간: [기간]
> 

> 
> 통계적 유의성 여부와 함께 실무 권고안을 제시해줘.

**활용 팁:** 통계 지식이 없어도 ChatGPT가 유의수준과 p-value 개념을 쉽게 풀어 설명해줍니다.

## 마케팅

### 8. [마케팅] 광고 카피 생성

**용도:** 제품·서비스 광고용 카피 다수 생성

**프롬프트:**

> [제품/서비스명]의 광고 카피를 5가지 스타일로 작성해줘.
> 
>   * 제품 특징: [핵심 특징 3가지]
>   * 타깃 고객: [연령대, 관심사 등]
>   * 채널: [SNS / 배너 / 검색광고 등]
>   * 카피 길이: [15자 이내 / 30자 이내 등]
> 

> 
> 각 카피에 사용한 심리적 트리거(희소성, 사회적 증거 등)도 설명해줘.

**활용 팁:** 마음에 드는 카피를 골라 "이 스타일로 10개 더 변형해줘"라고 이어가면 효율적입니다.

### 9. [마케팅] SNS 포스트 작성

**용도:** 인스타그램·X·링크드인 등 플랫폼별 맞춤 포스트 생성

**프롬프트:**

> 아래 내용을 바탕으로 [인스타그램 / X / 링크드인] 포스트를 작성해줘.
> 
>   * 포스트 주제: [신제품 출시 / 이벤트 안내 / 브랜드 스토리 등]
>   * 핵심 메시지: [전달하고 싶은 내용]
>   * 해시태그: [포함 여부 및 개수]
>   * 이모지 사용: [사용 / 미사용]
> 

> 
> 플랫폼 특성에 맞는 길이와 톤으로 작성해줘.

**활용 팁:** 링크드인은 전문적 톤, 인스타그램은 감성적 톤으로 자동 조정되도록 플랫폼을 명시하세요.

### 10. [마케팅] SEO 블로그 포스트 초안

**용도:** 검색 최적화된 블로그 글 초안 작성

**프롬프트:**

> 아래 조건으로 SEO 최적화된 블로그 포스트를 작성해줘.
> 
>   * 타깃 키워드: [메인 키워드], [서브 키워드 2~3개]
>   * 주제: [글의 핵심 주제]
>   * 타깃 독자: [초보자 / 전문가 / 구매 의향자 등]
>   * 분량: [약 1000자 / 1500자 / 2000자]
> 

> 
> H2·H3 제목 구조를 포함하고 키워드를 자연스럽게 배치해줘.

**활용 팁:** 글이 완성되면 "이 글에서 강조할 메타 디스크립션(160자)도 작성해줘"를 추가하세요.

## 코드

### 11. [코드] 버그 수정 요청

**용도:** 에러가 발생한 코드의 원인 분석 및 수정

**프롬프트:**

> 아래 코드에서 버그를 찾아 수정해줘.
> 
> `[언어명] [버그가 있는 코드 붙여넣기]`
> 
> 에러 메시지: [에러 메시지 붙여넣기]
> 
>   1. 버그 원인을 설명해줘.
>   2. 수정된 코드를 제시해줘.
>   3. 같은 실수를 방지하는 방법도 알려줘.
> 


**활용 팁:** 에러 메시지 없이 "예상과 다르게 동작한다"는 경우엔 기대 동작과 실제 동작을 함께 설명하세요.

### 12. [코드] 코드 리뷰 요청

**용도:** 작성한 코드의 품질·보안·성능 검토

**프롬프트:**

> 아래 코드를 리뷰해줘.
> 
> `[언어명] [리뷰할 코드 붙여넣기]`
> 
> 다음 관점에서 검토해줘: - 코드 가독성 및 네이밍 - 잠재적 버그 또는 엣지 케이스 - 성능 개선 포인트 - 보안 취약점
> 
> 수정 제안은 구체적인 코드 예시와 함께 제시해줘.

**활용 팁:** 팀 코딩 컨벤션(예: PEP 8, Google Style Guide)을 프롬프트에 명시하면 더 정확한 리뷰가 됩니다.

### 13. [코드] 기능 구현 요청

**용도:** 새로운 함수·모듈·기능 코드 생성

**프롬프트:**

> 다음 기능을 구현하는 [언어명] 코드를 작성해줘.
> 
>   * 기능 설명: [구현할 기능 상세 설명]
>   * 입력: [입력 데이터 형태 및 예시]
>   * 출력: [기대하는 출력 형태]
>   * 사용 가능한 라이브러리: [사용 가능/불가 라이브러리]
>   * 고려할 엣지 케이스: [빈 입력 / 중복 / 대용량 등]
> 

> 
> 코드에 독스트링과 주석을 포함해줘.

**활용 팁:** 코드가 생성되면 "이 함수에 대한 단위 테스트도 작성해줘"로 이어가면 TDD 사이클을 완성할 수 있습니다.

## 리서치

### 14. [리서치] 시장 조사 보고서 작성

**용도:** 특정 산업·시장의 현황과 트렌드를 조사·정리

**프롬프트:**

> [산업/시장명]에 대한 시장 조사 보고서를 작성해줘.
> 
> 다음 항목을 포함해줘: 1. 시장 규모 및 성장률 2. 주요 플레이어 및 경쟁 구도 3. 핵심 트렌드 3가지 4. 기회 요인과 위협 요인 5. 향후 전망
> 
> 각 항목에 구체적인 수치나 사례를 포함해줘.

**활용 팁:** ChatGPT의 학습 데이터 기준일 이후 최신 통계는 Browse 기능을 켜거나 직접 검색한 데이터를 붙여넣어 보완하세요.

### 15. [리서치] 문헌/자료 검색 및 요약

**용도:** 특정 주제의 핵심 정보를 체계적으로 수집·요약

**프롬프트:**

> [주제]에 대해 알아야 할 핵심 정보를 체계적으로 정리해줘.
> 
>   * 초보자도 이해할 수 있는 개념 설명
>   * 전문가들이 주목하는 핵심 논점 3가지
>   * 실무에서 자주 쓰이는 사례 2~3개
>   * 더 깊이 공부하려면 참고할 주요 자료/키워드
> 

> 
> 전체 분량은 500자 내외로 요약해줘.

**활용 팁:** 원문 자료(영어 논문, 보고서)를 붙여넣고 "한국어로 핵심만 요약해줘"라고 하면 번역+요약을 동시에 처리합니다.

## HR

### 16. [HR] 채용 공고 작성

**용도:** 포지션별 매력적인 채용 공고문 작성

**프롬프트:**

> 아래 조건으로 채용 공고를 작성해줘.
> 
>   * 포지션: [직무명]
>   * 주요 업무: [업무 내용 3~5가지]
>   * 자격 요건: [필수/우대 요건]
>   * 회사 특징: [회사 규모, 문화, 복지 포인트]
>   * 톤: [스타트업 감성 / 대기업 공식 / 친근한 등]
> 

> 
> 지원자가 지원하고 싶어질 만큼 매력적으로 작성해줘.

**활용 팁:** "MZ세대가 선호하는 워딩으로 바꿔줘"처럼 타깃 세대를 지정하면 더 효과적입니다.

### 17. [HR] 성과 피드백 작성

**용도:** 직원 성과 평가 시 건설적인 피드백 문서 작성

**프롬프트:**

> 아래 상황에 맞는 성과 피드백을 작성해줘.
> 
>   * 대상자: [직급/역할]
>   * 잘한 점: [구체적 성과나 행동]
>   * 개선이 필요한 점: [개선 영역]
>   * 다음 분기 목표: [설정할 목표]
> 

> 
> SBI(상황-행동-영향) 모델을 활용해서 구체적이고 건설적인 피드백으로 작성해줘.

**활용 팁:** "위 피드백을 1:1 면담 대화 스크립트로 변환해줘"라고 하면 실제 대화에도 활용할 수 있습니다.

## 번역/교정

### 18. [번역] 한영 전문 번역 및 현지화

**용도:** 비즈니스 문서의 전문적인 한영 번역

**프롬프트:**

> 아래 한국어 텍스트를 영어로 번역해줘.
> 
> [번역할 텍스트]
> 
>   * 문서 유형: [공식 문서 / 마케팅 자료 / 기술 문서 등]
>   * 타깃 독자: [미국 비즈니스 파트너 / 글로벌 고객 등]
>   * 유지할 브랜드 톤: [전문적 / 친근한 / 혁신적 등]
> 

> 
> 번역 후 어색한 표현이 있으면 자연스러운 대안도 함께 제시해줘.

**활용 팁:** "미국식 영어"와 "영국식 영어" 중 선호하는 스타일을 명시하면 더 일관된 결과를 얻습니다.

### 19. [교정] 문장 교정 및 가독성 향상

**용도:** 초안 문서의 맞춤법·문체·가독성 전반 교정

**프롬프트:**

> 아래 글을 교정해줘.
> 
> [교정할 텍스트]
> 
> 다음 사항을 중점적으로 확인해줘: - 맞춤법 및 띄어쓰기 오류 - 어색한 문장 구조 개선 - 중복 표현 제거 - 전체 문체 통일 ([격식체 / 비격식체])
> 
> 수정 사항은 원문과 대조하여 표로 정리해줘.

**활용 팁:** 전체 교정 후 "수정 이유를 각 항목별로 설명해줘"라고 하면 글쓰기 실력 향상에도 도움이 됩니다.

## 기획/전략

### 20. [전략] SWOT 분석 및 전략 도출

**용도:** 비즈니스·프로젝트의 전략적 상황 분석 및 전략 수립

**프롬프트:**

> [기업명/프로젝트명]에 대한 SWOT 분석을 수행하고 전략을 도출해줘.
> 
> 내가 알고 있는 정보: - 강점(Strength): [알고 있는 강점] - 약점(Weakness): [알고 있는 약점] - 현재 상황: [시장 환경, 경쟁자 정보 등]
> 
> 다음을 포함해서 완성된 SWOT 분석을 작성해줘: 1. S/W/O/T 각 항목별 3가지 요소 2. SO·ST·WO·WT 4가지 전략 방향 3. 가장 우선순위가 높은 전략 1가지 추천 및 이유

**활용 팁:** 경쟁사 정보나 뉴스 기사를 붙여넣으면 O(기회)와 T(위협) 항목의 완성도가 높아집니다.
