# XPLA 플랫폼 개발

## 프로젝트 개요

컴투스 그룹의 XPLA 블록체인 생태계에서, 여러 게임이 **SDK 기반으로 블록체인 기능을 사용**할 수 있도록 지원하는 공통 서비스 플랫폼. 게임 개발자가 블록체인 구현의 복잡도를 직접 부담하지 않고도, 지갑·NFT 민팅·트랜잭션 서명/전송 같은 핵심 기능을 일관된 방식으로 연동할 수 있게 한다.

- **기간**: 2024-04 ~ 2024-07 (4개월)
- **역할**: 시스템 설계 · API 정의 · 백엔드/프론트엔드(웹뷰) 개발 · 테스트 · 유지보수 · 리팩토링 주도
- **기술 스택**: Node.js (NestJS, TypeORM), React, TypeScript, Docker
- **선행 인프라**: 2022-07 "블록체인 웹 지갑 상용 DB 서버"(GCPHDBA-1245) 기반 위에서 확장

## 배경 (Situation)

- XPLA 체인 기반 게임이 늘어나며, 각 게임이 **지갑 연동·NFT 민팅·서명/전송**을 개별 구현하면 중복 비용과 품질 편차가 컸다.
- XPLA 고유의 수수료 산정·tx 서명·민팅 규칙을 일관되게 추상화할 **공통 플랫폼**이 필요했다.
- 2022년에 구축된 블록체인 웹 지갑 DB(`wallet-db-live-11/12` MASTER-SLAVE)가 이미 운영 중이어서, 그 위에 게임용 SDK 연동 레이어를 얹는 형태가 자연스러웠다.

## 목표 (Task)

1. **시스템 설계·API 정의**: SDK가 호출할 백엔드 API 계약과 웹뷰 프론트엔드의 사용자 흐름을 명세.
2. **핵심 기능 구현**: NFT Mint 플로우(Action 조회 → 정보/수수료 노출 → tx 생성 → 서명 → 전송 → 결과)를 E2E로 구현.
3. **엣지 케이스 처리**: 이미 민팅된 token ID 재요청, 수수료보다 부족한 XPLA 잔액 등 실사용 시 발생할 에러를 선제적으로 핸들링.
4. **품질 확보**: 리팩토링·버그 수정·다국어 확장성까지 포함해 출시 후 유지보수 비용을 최소화.

## 작업 내역 (Action)

### 1. 시스템 설계 & API 정의

- Node.js(NestJS, TypeORM)로 백엔드를 구성, XPLA 체인 호출·DB 영속 계층·SDK용 REST 인터페이스를 분리.
- 게임 SDK가 호출할 API 계약을 명확히 정의하여 게임팀과의 통합 비용을 최소화.
- React 기반 웹뷰(게임 클라이언트에서 임베드) 프론트엔드의 **페이지 단위 상태 전이**(민팅 정보 → 서명 → 전송 → 결과)를 설계.

### 2. NFT 민팅 플로우 구현 (CPBLOC-821)

민팅 1건에 대해 다음 전체 흐름을 단일 UX로 묶었다:

1. Action 조회 및 정보 노출
2. 수수료 정보 노출
3. NFT 민팅 tx 생성
4. 서명
5. tx 전송 페이지로 이동하여 tx 전송
6. 결과 페이지로 이동하여 결과 노출

에지 케이스:

- 이미 민팅된 token ID 요청 → 민팅 불가 안내로 분기
- 총 수수료 > 보유 XPLA → 잔액 부족 가드 후 요청 중단

### 3. 리팩토링 & 버그 수정 (CPBLOC-861)

- 클린 코드 원칙(명확한 네이밍·단일 책임·중복 제거)에 따라 모듈을 정리.
- 개발 중 노출된 버그를 수정하며 코드 품질을 출시 수준으로 끌어올림.

### 4. 웹뷰 다국어 기반 작업 (CPBLOC-862, 요청자)

- 다국어 라이브러리를 추가하고 샘플 문구로 동작 검증.
- 초기에는 영어만 지원하되, **URL 언어 코드(en/ko) 기반**으로 즉시 한글 확장이 가능한 구조를 마련.
- 본 작업은 후속 유지보수 단계에서 동료(유인환)가 전체 문구 적용을 수행할 수 있도록 기반을 준비한 것.

### 5. 배포/운영

- Docker 기반으로 배포 환경을 일관화.
- 2022년 구축한 `wallet-db-live-*` (MySQL 8 MASTER-SLAVE) 인프라 위에서 안정적으로 운영.

## 결과 (Result)

- **일정 준수**: 2024-04 ~ 2024-07 일정 내 XPLA 플랫폼 개발 완료.
- **게임 접근성 개선**: 게임 개발팀은 블록체인 구현 세부사항(tx 생성·서명·수수료) 없이 SDK 호출만으로 지갑·민팅 기능을 통합 가능.
- **코드 품질**: 리팩토링·엣지 케이스 처리·다국어 확장성 확보로 출시 후 유지보수 비용을 최소화.
- **티켓 완수**: CPBLOC 클러스터 19건 전수 Done 처리(Phase B 수집 기준). 이후 다국어 적용(2024-08-27)까지 후속 작업이 자연스럽게 이어짐.
