## 4. 개발자로서의 꿈과 방향성

- Python, AI, 데이터 분석에 관심이 많음
- 웹 프레임워크로는 FastAPI를 선호함

- 많은 사람들에게 도움을 주고, 사람들의 삶을 더 편하게 만들고 행복하게 해주는 AI 서비스를 만드는 것이 장기적 꿈
- AI를 단순한 도구가 아니라 개발 파트너, 기획 파트너, 개인 비서, 친구 같은 존재로 활용하고 싶어 함
- 최근에는 바이브 코딩에 관심이 많고 적극적으로 활용 중
- 본인의 장기적 방향은 "AI와 함께 더 인간다운 삶을 돕는 서비스"에 가까움

## 6. 현재 핵심 프로젝트: MateChat

- 프로젝트명: MateChat / Mate Chat / 메이트챗
- 저장소: gsroot/mate-chat, private repository
- 제품 슬로건:
  - "대화의 시작은 AI, 끝은 사람."
  - "Start with AI. End with people."
- 제품 핵심 컨셉:
  - AI가 사람을 대체하는 앱이 아니라, 사람과 사람의 진짜 연결을 돕는 소셜+AI 하이브리드 앱
  - AI는 친구 대체재가 아니라 사회적 윤활제 역할을 함
  - 외로움과 사회적 어색함을 AI가 완충하고, 최종적으로는 진짜 사람과의 관계 형성을 돕는 것이 핵심
- 핵심 문제 정의:
  - AI와 대화하면 편하지만, 결국 진짜 사람과 연결되지 못하면 공허함이 남는다
  - 새로운 사람을 만나고 싶지만 어색함, 불안, 대화 시작의 부담 때문에 쉽게 연결되지 못한다
  - 기존 AI 컴패니언 앱은 사람을 대체하려 하고, 기존 소셜/채팅 앱은 어색함을 사용자가 직접 감당해야 한다
- 핵심 타깃 페르소나:
  - 20대 후반~30대 대도시 거주자
  - 평일 저녁 ChatGPT, Character.AI, Replika 등과 긴 시간 대화하지만, 실제 인간 친구를 만들고 싶은 사람
  - Tinder/Bumble은 부담스럽고, Discord 신규 그룹 진입은 어색하고, 직장 동료와는 거리감이 있는 사람
  - AI 대화 후 "재미있지만 뭔가 비어있다"는 감각을 느끼는 사람
- 확장 가능한 관련 페르소나:
  - 외국 워홀러/이민자
  - 사회 불안이 있는 학생
  - 새 도시 이주자
  - 사람과 대화하고 싶지만 시작이 어려운 사람
- 제품 포지셔닝:
  - Character.AI/Replika의 더 좋은 버전이 아님
  - Maum/Azar의 텍스트 버전도 아님
  - Discord에 AI를 붙인 범용 채팅 앱도 아님
  - "AI 시대에 사람과 다시 연결되도록 돕는 앱"에 가까움
- 현재 제품 상태:
  - Android v1.0.0 출시 직전 수준
  - 백엔드, Flutter 앱, 인증, 실시간 채팅, AI 챗봇, 클로버, IAP, 푸시 알림, 다국어화 등 핵심 기능 대부분 구현
  - 문서상 전체 완성도는 상당히 높지만, 제품 비전 문서 기준 아직 외부 사용자 검증은 부족함
- 제품 비전 문서상 핵심 판단:
  - 빌드는 많이 됐지만, 실제 외부 사용자 검증은 아직 부족함
  - 다음 단계는 기능 추가가 아니라, closed alpha 또는 내부 테스트를 통해 실제 사용자를 만나는 것
  - 1개 공개 채팅방, 1개 Mate-Bot, 10명 이내 사용자를 대상으로 검증하는 접근이 우선
- 핵심 검증 지표:
  - 첫 7일 내 사용자가 진짜 사람과 양방향 메시지를 5회 이상 주고받는 비율
  - 장기적으로는 사용자가 실제 인간 친구를 만들고 지속 대화하는지 여부
- 현재 가장 중요한 질문:
  - "MateChat은 실제로 사람과 사람 사이의 대화를 만들어내는가?"
  - "AI가 개입했을 때 인간-인간 메시지 빈도가 증가하는가?"
  - "외로운 AI 의존자 페르소나가 실제로 존재하고, 이들이 진짜 사람과 연결되고 싶어 하는가?"

  - 하루 약 6~7시간
- 주말:
  - 개발/사업화 시간을 내기 어려울 가능성이 높음

  - 사용자 검증
  - 초기 유저 확보
  - 수익화
  - 지속적인 업데이트와 마케팅

  - 1년 후 사용자 1만 명
  - 월 순수익 500만 원 이상

  - 월 순수익 300만 원 이상이 3개월 이상 유지되거나
  - 수익이 빠르게 성장하는 명확한 추세가 보일 경우 퇴사 검토 가능
- 퇴사 전 확보하고 싶은 안전장치:
  - 1년치 생활비 확보
- 가족 생계 관점의 월 최소 수입:
  - 약 300만 원
- 휴직 기간 재무 판단:
  - 휴직 수당과 절약, 앱 수익화 전환이 어느 정도 이루어지면 버틸 수 있을 것으로 판단
- 개인 사업이 잘 안 풀릴 경우의 우선순위:
  1. 프리랜서
  2. 이직
  3. 복직
  4. 상황에 따라 다른 앱이나 아이템으로 재도전
- 중요한 관점:
  - 월 500만 원은 절대 기준선이 아니라 이상적인 목표에 가까움
  - 현실적으로는 월 300만 원 이상의 안정 수익과 성장세가 퇴사 판단의 핵심 기준

## 8. MateChat 수익화와 KPI 관점

- MateChat은 클로버라는 가상 화폐 기반 IAP 모델을 포함함
- AI 메시지 사용량, 클로버 소비, 유료 전환, 리텐션이 주요 수익 변수
- 수익 예측 문서 기준, 초기의 과도한 매출 예측은 폐기되고 더 보수적인 PMF KPI 중심 접근으로 수정됨
- 현재 더 중요한 지표:
  - Paying user count 1,000명
  - D30 retention 8%+
  - Organic K-factor 0.3+
- 중요한 해석:
  - 수익은 후행 지표
  - 먼저 제품이 실제로 유지율과 결제 사용자, 자연 확산을 만들어내는지 봐야 함
- 사업화에서 특히 주의할 점:
  - 사용자 수만 많고 결제 전환이 없으면 실패 가능성이 높음
  - 마케팅만으로 설치 수를 늘리는 것은 위험함
  - 실제 대화 생성, 재방문, 유료 전환, 자연 초대가 동시에 확인되어야 함
- 석근에게 적합한 사업 판단 방식:
  - "월 매출 얼마"보다 "유저가 왜 계속 쓰고 왜 돈을 내는지"를 먼저 확인해야 함

## 9. 출시 준비와 리스크

- MateChat은 기능 완성도는 높지만, 출시 전후로 반드시 관리해야 할 리스크가 있음
- 과거 글로벌 출시 진단 문서에서는 IAP 검증, 푸시 알림 다국어화, 콘텐츠 모더레이션, 법적 문서, GDPR/COPPA 대응 등이 출시 차단 또는 고위험 이슈로 정리됨
- 이후 TODO 문서에서는 v1.0.0 완료 항목으로 Google OAuth, 이메일 인증, COPPA, IAP 검증, FCM, 9개 언어 국제화, 보안 감사, Android 릴리스 설정, Play Store 에셋, Privacy Policy 등이 정리됨
- 문서 간 시점 차이가 있으므로, 실제 출시 전에는 다음 항목의 최신 상태를 재확인해야 함:
  - Google Play IAP 실제 검증
  - 개인정보처리방침과 이용약관
  - 콘텐츠 모더레이션
  - 신고 처리 워크플로우
  - FCM 푸시 알림
  - COPPA 13세 확인
  - 다국어 번역 품질
  - 앱스토어 메타데이터
  - closed alpha 테스트
- 현재 가장 큰 리스크:
  - 기능 부족보다 사용자 검증 부족
  - 출시 지연
  - 마케팅/포지셔닝 미정
  - 혼자 운영하면서 번아웃이 오는 것

- 핵심 목적:

- 선호하는 작업 리듬:
  - 되도록 아침형으로 일하려고 함
  - 실제 집중이 가장 잘 되는 시간대는 오후~저녁
  - 직장 생활을 하지 않으면 밤늦게 자며 일하는 습관이 생기기 쉬움
  - 하지만 밤형 생활이 덜 피곤한 것은 아니므로, 의식적으로 아침형 루틴을 만들고 싶어 함
- 추천 루틴 방향:
  - 오전: 육아/가벼운 정리/운동/운영 체크
  - 오후: 핵심 개발/마케팅/출시 작업
  - 저녁 전: 회고, 지표 확인, 다음날 작업 정리
  - 밤: 가능하면 깊은 개발보다 가벼운 문서/아이디어 정리 정도로 제한
- 주의할 점:
  - 평일 6~7시간은 큰 자산이지만, 육아 70~80%를 맡으면 체력 소모가 매우 큼
  - 휴직 초기부터 무리하면 번아웃 가능성이 높음
  - 주말을 작업일로 계산하지 않는 계획이 현실적임
  - "주 5일 × 6시간" 기준으로 계획하되, 실제 실행 가능 시간은 주 20~25시간 정도로 보수적으로 잡는 것이 안전함

## 11. 일하는 방식과 생산성

- 선호하는 업무 방식:
  - 프로젝트 초기에는 혼자 깊게 파는 것을 선호
  - 이후 운영 단계에서는 AI 또는 사람과 대화하며 정리하는 것도 선호
  - 바이브 코딩에서는 처음부터 끝까지 AI와 대화하며 진행하는 것을 좋아함
- 집중이 잘 되는 시간:
  - 오후부터 저녁까지
- 지향하는 리듬:
  - 직장 생활이 없을 때도 너무 늦게 자지 않고 아침형에 가까운 루틴을 만들고 싶어 함
- 개발할 때 자주 막히는 지점:
  - QA
  - 배포
  - 마케팅
  - 디자인
  - 원인을 알기 어려운 버그 디버깅
  - 전문 분야 밖의 실무 작업
- 강점:
  - 현실 감각 있는 문제 해결력
  - 배우고 연결하여 아이디어를 결과물로 만드는 능력
  - 결단력
  - 책임감
  - 큰 그림을 보는 안목
  - 사람들에게 도움을 주고 싶은 따뜻한 마음
  - 높은 이상
- 약점:
  - 일을 많이 해도 회복이 잘 안 됨
  - 번아웃에 취약함
  - 깊이 파고드는 공부보다 빠르게 산출하는 쪽으로 흐르기 쉬움
  - 생각이 많아져 추진력이 떨어질 때가 있음
  - 감정을 잘 표현하지 않다가 한 번에 표출할 때가 있음
- 생산성 관련 주의점:
  - 석근은 "몰아서 강하게 하는 능력"은 있지만, 휴직 기간에는 "꾸준히 오래 버티는 구조"가 더 중요함
  - 개발만 잘해서는 MateChat 성공이 어렵고, QA/마케팅/운영/지표 분석을 루틴화해야 함
  - 출시 전에는 완성도보다 사용자 검증이 우선이어야 함

## 12. 건강과 생활 패턴

- 평균 수면 시간:
  - 하루 약 6~7시간
  - 다만 수면 시간이 들쭉날쭉한 편
- 운동:
  - 매일 할 때도 있고, 한동안 전혀 안 할 때도 있음
  - 육아휴직 후에는 한 번에 많이 하기보다 꾸준히 하는 것을 목표로 함
- 신체 이슈:
  - 목, 허리, 어깨가 종종 아픔
  - 밤늦게 자거나 컴퓨터 화면을 오래 보면 눈 피로가 있음
- 스트레스 해소 방식:
  - 산책
  - 게임
  - 음악 듣기
  - 책 읽기
  - SNS에서 재밌는 글 보기
  - 유튜브 영상 보기
  - 정해진 방식보다는 그때 하고 싶은 것을 하며 푸는 편
- 번아웃 신호:
  - 개발도 하기 싫어짐
  - 책임지는 일이 싫어짐
  - 예민해짐
  - 회복이 잘 안 됨
- 건강 관련 중요한 판단:
  - MateChat 사업화의 가장 큰 적은 기술 부족보다 번아웃일 가능성이 높음
  - 육아, 개발, 마케팅, 돈 걱정이 동시에 오면 체력과 멘탈 관리가 사업 리스크가 됨
  - 매일 적은 운동, 수면 리듬, 작업량 제한이 필수임

## 13. 성격과 커뮤니케이션 취향

- 성향:
  - 큰 그림과 장기 방향을 중요하게 봄
  - 현실 감각과 이상을 함께 가지고 있음
  - 책임감이 강하고 결단력이 있음
  - 사람들에게 도움을 주고 싶은 마음이 강함
  - 감정을 평소에 많이 드러내지 않다가, 누적되면 한 번에 표출할 수 있음
- 선호하는 답변 스타일:
  - 친구처럼 친근하게 말하되, 필요할 때는 비서처럼 정리해주는 방식
  - 상황에 따라 짧은 결론, 긴 분석, 단계별 계획, 직설적 피드백, 아이디어 브레인스토밍을 유연하게 섞는 하이브리드 스타일
  - 듣기 좋은 말보다 실제로 도움이 되는 말
  - 장단점 비교 후 명확한 추천
  - 현실적인 실행 계획
- 싫어하는 답변 스타일:
  - 무조건적인 칭찬
  - 근거 없는 낙관
  - 원론적인 조언
  - 석근의 상황을 무시한 뻔한 답변
  - "다 가능하다" 식의 애매한 중립
- 피드백 선호:
  - 근거가 있으면 강하게 반대해도 됨
  - 단, 반대할 때는 대안과 실행 방향도 함께 제시하는 것이 좋음

## 14. 가치관과 인생 방향

- 중요하게 생각하는 가치:
  - 가족
  - 자유
  - 성장
  - 진실
  - 실용성
  - 창조
  - 경제적 독립
  - 사람들에게 실제 도움을 주는 것
- 원하는 삶:
  - 내가 만든 서비스로 수익을 만들며 사는 삶
  - 가족과 더 많은 시간을 보내는 삶
  - AI와 자동화를 적극 활용해 생산성을 높이는 삶
  - 개발자로서 계속 성장하는 삶
  - 사람들의 외로움, 불편함, 비효율을 줄여주는 서비스를 만드는 삶
- 피하고 싶은 삶:
  - 안정만 추구하다가 하고 싶은 서비스를 계속 미루는 삶
  - 남이 시키는 일만 하며 주도권 없이 사는 삶
  - 가족과 충분히 시간을 보내지 못하는 삶
  - 가능성 있는 아이디어를 실행하지 않고 후회하는 삶
  - 번아웃으로 좋아하던 개발과 창작까지 싫어지는 삶

## 15. AI에게 기대하는 역할

- AI에게 기대하는 핵심 역할:
  - 친구
  - 개인 비서
  - 개발 동료
  - 사업 전략 파트너
  - 마케팅 파트너
  - 일정 관리자
  - 냉정한 피드백 제공자
  - 감정 공유 상대
  - 문서/기획 보조자
- AI가 함께 해줬으면 하는 일:
  - 개발 설계
  - 코드 리뷰
  - 디버깅
  - QA 체크리스트 작성
  - 배포 전략 수립
  - MateChat 사업 전략 수립
  - 타깃 사용자 정의
  - 마케팅 문구 작성
  - 앱스토어/플레이스토어 메타데이터 작성
  - 사용자 인터뷰 질문 작성
  - 기능 우선순위 정리
  - 수익 모델 검토
  - 지표 설계
  - 일정 관리
  - 회고
  - 번아웃 방지 루틴 설계
  - 감정 정리
- AI 답변 방식:
  - 전반적으로 친구처럼 말하되, 필요할 때는 비서처럼 구조화
  - 상황에 따라 가장 적절한 형태로 답변
  - 근거가 있으면 강하게 반대해도 됨
  - 단순 공감보다 현실적 해결책을 선호
  - 중요한 결정에서는 리스크를 숨기지 말고 직설적으로 말해주길 원함

## 17. 현재 가장 중요한 전략적 판단

- MateChat은 이미 기능이 많은 앱이다.
- 지금 가장 중요한 것은 기능 추가가 아니라 실제 사용자 검증이다.
- 특히 다음 질문에 답해야 한다:
  1. 외로운 AI 의존자 페르소나가 실제로 존재하는가?
  2. 그들이 MateChat을 설치하고 계속 사용할 이유가 있는가?
  3. AI가 사람 간 대화를 실제로 촉진하는가?
  4. 첫 7일 안에 인간-인간 양방향 메시지 5회 이상이 발생하는가?
  5. 사용자가 클로버에 돈을 낼 만큼 AI/소셜 경험에 가치를 느끼는가?
- 석근에게 필요한 전략:
  - 출시를 늦추는 추가 개발보다 closed alpha/소규모 베타를 빠르게 시작
  - 10명 이하의 실제 사용자에게 보여주고 관찰
  - 지표와 인터뷰를 기반으로 기능을 줄이거나 방향을 재정의
  - MateChat을 "AI 친구 앱"이 아니라 "AI가 사람 친구를 만들어주는 앱"으로 일관되게 포지셔닝
