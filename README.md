# LLM Wiki

LLM 에이전트가 점진적으로 구축하고 유지관리하는 개인 종합 위키.

## 개요

RAG처럼 매번 원시 문서에서 검색하는 대신, LLM이 소스를 읽고 핵심을 추출하여 **영속적인 위키에 통합**합니다. 상호 참조는 미리 만들어져 있고, 모순은 기록되어 있으며, 소스를 추가할수록 위키는 복리로 성장합니다.

- **Obsidian** — 위키 뷰어 (그래프 뷰, Dataview, 위키링크)
- **Claude Code / Cowork** — 위키 작성·관리 에이전트
- **Git** — 수집 단위 커밋으로 변경 이력 추적

## 구조

```
llm-wiki/
├── CLAUDE.md          # 스키마 (운영 규칙, 워크플로우)
├── raw/               # 원시 소스 (불변, 읽기 전용)
│   ├── articles/      # 웹 기사, 블로그
│   ├── papers/        # 논문, 보고서
│   ├── notes/         # 메모, 회의록
│   └── assets/        # 이미지, PDF
├── wiki/              # LLM이 관리하는 위키
│   ├── entities/      # 사람, 조직, 도구
│   ├── concepts/      # 개념, 기술, 방법론
│   ├── sources/       # 소스별 요약
│   ├── syntheses/     # 종합 분석
│   ├── logs/log.md    # 활동 로그
│   └── index.md       # 전체 카탈로그
└── templates/         # 페이지 템플릿
```

## 사용법

### 소스 추가 (수집)

1. `raw/` 하위 폴더에 소스 파일을 넣는다
2. LLM에게 "수집해줘"라고 지시한다
3. LLM이 소스 요약, 엔티티/개념 페이지 생성·업데이트, 인덱스·로그 갱신을 수행한다

### 질문하기 (질의)

위키에 대해 자유롭게 질문하면, LLM이 인덱스에서 관련 페이지를 찾아 종합 답변을 제공한다.

### 상태 점검 (Lint)

"점검해줘"라고 요청하면 깨진 링크, 고아 페이지, 모순, 빈약한 페이지 등을 확인하고 개선안을 제시한다.

## 규칙

- `raw/` 파일은 **절대 수정하지 않는다**
- `wiki/` 파일은 **LLM만 작성/수정**한다
- 파일명은 영어 `kebab-case`, 제목은 한국어
- 모든 위키 페이지는 YAML 프론트매터 필수

자세한 운영 규칙은 [CLAUDE.md](CLAUDE.md)를 참고.
